# 修改文档

修改已存在的文档文件，保存新版本并记录修改历史。

## 触发条件

- 需要修改已存在的文档文件时
- 需要保存文档新版本时
- 需要记录文档修改历史时

## 执行方式

主代理执行

## 依赖

- 项目已存在
- 目标文件路径有效

## 输入

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 项目名称 | string | 是 | 目标项目名称 |
| 文件路径 | string | 是 | 相对于项目根目录的路径 |
| 修改内容 | string | 是 | 完整的新文件内容或差异修改 |
| 版本升级 | string | 否 | major/minor/patch，默认patch |

## 步骤

### Step 1: 确认信息
- 确认项目名称和文件路径
- 确认修改内容
- 确认版本升级方式（major/minor/patch，默认patch）

### Step 2: 读取当前版本
- 从项目元数据读取当前版本号
- 如果没有则初始化为 v1.0.0

### Step 3: 计算新版本号
- `major`: 主版本+1，次版本=0，修订=0 (v1.0.0 → v2.0.0)
- `minor`: 主版本不变，次版本+1，修订=0 (v1.0.0 → v1.1.0)
- `patch`: 主版本不变，次版本不变，修订+1 (v1.0.0 → v1.0.1)

### Step 4: 备份旧版本
- 创建备份目录 `草稿/{filename}/`（如果不存在）
- 复制当前文件到备份目录
- 命名格式：`{filename}_v{old_version}.md`

### Step 5: 创建新文件
- 在路径中创建 `{filename}.md`，不要添加版本号到文件名
- 写入修改后的内容
- 验证文件可读取

### Step 7: 验证
- 检查文件内容是否正确
- 检查备份是否存在
- 检查版本号是否正确更新

### Step 8: 记录到工作日志
- 记录修改时间、项目、文件路径、版本变化

## 输出

- 修改后的完整文件路径
- 旧版本已备份到 `草稿/{filename}/`
- 新版本号
