# 撰写脚本

> 版本: 1.0.0  
> 维护者: 大管家  
> 创建时间: 2026-04-08

## 功能描述

本脚本帮助用户按照五要素SOP规范撰写新的操作脚本。输出包含核心文件、README和SKILL三个文件内容，但不涉及实际存储操作。

## 使用方法

1. **确认脚本需求**：明确脚本名称、功能和触发条件
2. **撰写核心脚本**：按照五要素结构（触发条件、依赖、输入、步骤、输出）撰写
3. **创建配套文件**：
   - `README.md`：给人类看的说明文档
   - `SKILL.md`：给AI看的结构化技能说明
4. **检查一致性**：确保三个文件内容一致

## 文件结构

```
撰写脚本/
├── 撰写脚本.md    # 核心脚本（五要素SOP）
├── SKILL.md       # 给AI看的技能说明
└── README.md      # 给人类看的说明（本文件）
```

## 五要素结构

每个脚本必须包含以下五个要素：

| 要素 | 说明 | 示例 |
|------|------|------|
| 触发条件 | 什么时候需要运行这个脚本 | 需要创建新脚本时 |
| 依赖 | 依赖哪些其他脚本或技能 | 无 / 修改文档.md |
| 输入 | 需要哪些输入参数 | 脚本名称、功能描述 |
| 步骤 | 具体的执行步骤 | Step 1, Step 2... |
| 输出 | 脚本会输出什么 | 三个文件内容 |

## 配套文件说明

### README.md（给人类看）
- 使用自然语言描述
- 侧重可读性和易理解性
- 包含功能描述、使用方法、文件结构

### SKILL.md（给AI看）
- 使用结构化格式
- 包含YAML frontmatter元数据
- 便于AI解析和执行

## 注意事项

- 本脚本只输出文件内容，不涉及实际存储
- 三个文件内容应保持一致，只是呈现方式不同
- 步骤标题建议采用动宾结构，明确动作链条
