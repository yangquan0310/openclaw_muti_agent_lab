# 通用脚本技能

## 触发条件

需要执行标准化的文档管理、项目维护、日志记录等操作时。

## 执行方式

主代理执行

## 依赖

无

## 输入

根据具体脚本需求提供相应参数。

## 步骤

根据任务类型选择对应的脚本执行。

## 输出

根据具体脚本返回相应结果。

## 脚本清单

### 1. 修改文档

**文件**：`修改文档.md`

**功能**：修改已存在的文档文件，保存新版本并记录修改历史

**触发条件**：需要修改已存在的文档文件

**输入**：
- 项目名称：目标项目
- 文件路径：相对于项目根目录的路径
- 修改内容：完整的新文件内容（或差异修改）
- 版本升级：主版本号/次版本号/修订号（默认修订号+1）

**步骤**：
1. 确认项目名称和文件路径
2. 读取当前版本
3. 计算新版本号（major/minor/patch）
4. 备份旧版本到 `草稿/{filename}/`
5. 创建新文件（不添加版本号到文件名）
6. 更新项目元数据
7. 验证文件内容
8. 记录到工作日志

**输出**：
- 修改后的完整文件路径
- 旧版本备份位置
- 新版本号
- 修改记录

---

### 2. 撰写 Python 脚本

**文件**：`撰写py.md`

**功能**：创建 Python 脚本完成任务

**触发条件**：需要创建 Python 脚本完成任务时

**输入**：
- 任务描述：需要完成的任务目标
- 功能需求：脚本需要实现的具体功能
- 输入输出：脚本的输入参数和输出结果

**步骤**：
1. 分析任务要求
2. 建立面向对象编程框架
3. 撰写 Python 脚本
4. 存储脚本到 `scripts/<任务>/` 文件夹
5. 撰写 SKILL.md
6. 更新个人技能索引
7. 验证脚本可执行

**输出**：
- Python 脚本文件
- 对应的 SKILL.md 文档
- 个人技能索引更新

---

### 3. 撰写脚本

**文件**：`撰写脚本.md`

**功能**：按照五要素 SOP 规范撰写新脚本

**触发条件**：需要创建新的操作脚本

**输入**：
- 脚本名称
- 脚本功能描述
- 触发场景

**步骤**：
1. 确认脚本需求
2. 撰写核心脚本文件（五要素结构）
3. 撰写 README.md（给人类看）
4. 撰写 SKILL.md（给 AI 看）
5. 检查一致性

**输出**：
- 核心脚本文件内容
- README.md 内容
- SKILL.md 内容

**文件**：`撰写技能.md`

**功能**：创建新的技能（结构化或非结构化）

**触发条件**：需要创建新的技能

**输入**：
- 技能名称
- 技能功能描述
- 触发场景

**步骤**：
1. 确认技能需求
2. 撰写核心技能文件
3. 撰写 README.md（给人类看）
4. 撰写 SKILL.md（给 AI 看）
5. 检查完整性

**输出**：
- 核心技能文件内容
- README.md 内容
- SKILL.md 内容

### 5. 检索文献

**文件**：`检索文献.md`

**功能**：使用 Zotero 和 Semantic Scholar 检索学术文献

**触发条件**：需要检索学术文献时

**输入**：
- 检索关键词
- 检索来源（Zotero/Semantic Scholar/两者）
- 检索数量限制

**步骤**：
1. 选择检索工具
2. 执行检索
3. 整理结果
4. 输出文献列表

**输出**：
- 文献列表（标题、作者、年份、摘要等）

---

### 6. 管理项目元数据

**文件**：`管理项目元数据.md`

**功能**：创建和维护项目元数据，管理本地文档与云文档映射

**触发条件**：需要创建新项目元数据或维护云文档映射

**输入**：
- 项目ID
- 项目标题
- 创建日期
- 项目描述
- 状态（active/completed）

**步骤**：
1. 确定项目路径
2. 写入 `元数据.json`
3. 验证 JSON 格式

**输出**：
- 创建完成的 `元数据.json` 文件路径

---

### 7. 记录工作日志

**文件**：`记录工作日志.md`

**功能**：记录任务执行日志

**触发条件**：任务执行完成

**输入**：
- 任务信息
- 执行结果
- 参与 Agent

**步骤**：
1. 确定任务类型
2. 填写任务描述
3. 记录执行结果
4. 填写涉及项目
5. 记录存储路径
6. 列出参与 Agent
7. 详细说明分工
8. 添加备注
9. 保存日志文件

**输出**：
- 日志文件（路径：`~/实验室仓库/日志文件/YYYY-MM-DD/HH-MM-SS-{AGENT}-{任务类型}.md`）
