#### 管理项目元数据

##### 触发条件
需要创建新项目元数据，维护本地文档，维护本地文档与云文档（飞书/腾讯文档）的映射关系

##### 依赖
- 目标项目已创建（对于映射云文档）

##### 输入

###### 创建元数据时：
- 项目ID：项目名称
- 项目标题：项目标题
- 创建日期：创建日期（YYYY-MM-DD）
- 项目描述：项目描述
- 状态：active / completed

###### 元数据结构
```json
{
  "project_id": "项目名称",
  "title": "项目标题",
  "created_date": "创建日期",
  "status": "active",
  "version": "v1",
  "description": "项目描述",
  "directories": {},
  "files": [],
  "tags": [],
  "tasks": [],
  "cloud_doc_mappings": {}
}
```

###### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `project_id` | string | 是 | 项目唯一标识，通常与文件夹名一致 |
| `title` | string | 是 | 项目标题，用于展示 |
| `created_date` | string | 是 | 创建日期，格式：YYYY-MM-DD |
| `status` | string | 是 | 项目状态：`active` 或 `completed` |
| `version` | string | 是 | 当前版本号，格式：v1, v2... |
| `description` | string | 否 | 项目描述，说明项目目标和内容 |
| `directories` | object | 否 | 项目目录结构记录 |
| `files` | array | 否 | 项目文件列表 |
| `tags` | array | 否 | 项目标签，用于分类和检索 |
| `tasks` | array | 否 | 项目任务记录 |
| `cloud_doc_mappings` | object | 否 | 云文档映射关系，键为文档标题 |

###### cloud_doc_mappings 结构
```json
{
  "文档标题": {
    "local_path": "文档标题.md",
    "cloud": [
      {
        "platform": "feishu",
        "cloud_url": "https://...",
        "cloud_id": "doc-xxx",
        "created_at": "ISO时间戳",
        "updated_at": "ISO时间戳"
      }
    ]
  }
}
```

###### cloud_doc_mappings 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `local_path` | string | 是 | 本地文件相对项目根目录的路径 |
| `cloud` | array | 是 | 云文档平台列表 |
| `platform` | string | 是 | 平台名称：`feishu` 或 `tencent` |
| `cloud_url` | string | 是 | 云文档访问URL |
| `cloud_id` | string | 否 | 云文档ID |
| `created_at` | string | 是 | 创建映射时间，ISO格式 |
| `updated_at` | string | 是 | 最后更新时间，ISO格式 |

##### 步骤

1. **确定项目路径**
   - 项目路径：`~/实验室仓库/项目文件/项目名/`
   - 检查路径是否存在，如果不存在则提示错误

2. **写入元数据.json**
   - 确认项目名称
   - 确认本地文件路径
   - 确认云文档URL和平台
   - 写入到项目根目录
   - 设置正确文件权限：644

3. **验证**
   - 验证JSON格式是否正确

##### 输出
- 创建完成的 `元数据.json` 文件路径（创建时）
- 更新后的项目元数据.json（修改时）
