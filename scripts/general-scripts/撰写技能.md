# 撰写技能

## 触发条件
需要创建新的技能

## 输入
- 技能名称
- 技能功能描述
- 触发场景

## 步骤

1. **确认技能需求**
   - 明确技能命名
   - 明确技能功能
   - 明确触发条件

2. **撰写核心技能文件**
   
   如果是结构化技能（包含代码）：
   ```
   skills/{技能名}/
   ├── *.py              # Python脚本
   ├── *.sh              # Shell脚本
   ├── README.md         # 技能说明（给人类看）
   └── SKILL.md          # 技能规范（给AI看）
   ```
   
   如果是非结构化脚本（纯Markdown）：
   ```
   scripts/{脚本名}/
   ├── *.md              # 脚本内容
   ├── README.md         # 脚本说明（给人类看）
   └── SKILL.md          # 技能规范（给AI看）
   ```

3. **撰写 README.md**
   
   README.md 是给**人类**看的技能说明，包含：
   
   - **技能概述**
     - 技能名称和版本
     - 一句话功能描述
   - **功能描述**
     - 详细说明技能的功能模块
     - 使用表格列出子任务（如适用）
   - **使用方法**
     - 执行命令示例
     - 参数说明
   - **文件结构**
     - 列出文件夹内的文件及其作用
   - **版本历史**
     - 版本号、日期、更新内容

4. **撰写 SKILL.md**
   
   SKILL.md 是给**AI**看的技能使用说明，采用结构化格式：
   
   - **技能元数据**（YAML frontmatter）
     ```yaml
     ---
     name: 技能名称
     description: >
       技能描述
     metadata:
       openclaw:
         emoji: "🔄"
         requires:
           bins: ["bash"]
     ---
     ```
   - **触发条件**
     - 明确什么情况下触发此技能
   - **执行方式**
     - 子代理执行 / 主代理执行
   - **依赖**
     - 列出依赖的工具或环境
   - **输入**
     - 列出需要的输入参数
   - **步骤**
     - 详细的执行步骤
   - **输出**
     - 列出执行结果
   - **定时任务配置**（如适用）
     - 执行时间、任务ID、任务名称

5. **检查完整性**
   - 检查技能文件结构
   - 检查三个文件的一致性

## 输出

### 1. 核心技能文件内容

根据技能类型：
- **结构化技能**：Python/Shell 脚本代码
- **非结构化脚本**：Markdown 格式的脚本内容

### 2. README.md 内容

给人类看的技能说明：
- 技能概述（名称、版本、功能）
- 功能描述（详细说明，可使用表格）
- 使用方法（命令示例）
- 文件结构（文件列表）
- 版本历史

### 3. SKILL.md 内容

给 AI 看的技能使用说明：
- 技能元数据（YAML frontmatter）
- 触发条件
- 执行方式
- 依赖
- 输入
- 步骤
- 输出
- 定时任务配置（如适用）

**注意**：
- README.md 侧重**可读性**，使用自然语言描述
- SKILL.md 侧重**结构化**，便于 AI 解析执行
- 两个文件的内容应保持一致，只是呈现方式不同

### 4. 不涉及文件存储

仅生成文件内容，不实际创建文件
