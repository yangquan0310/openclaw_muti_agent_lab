# 撰写技能

> 版本: 1.0.0  
> 维护者: 大管家  
> 创建时间: 2026-04-08

## 功能描述

本脚本帮助用户创建新的技能，包括结构化技能（含代码）和非结构化脚本（纯Markdown）。输出包含核心文件、README和SKILL三个文件内容，但不涉及实际存储操作。

## 使用方法

1. **确认技能需求**：明确技能名称、功能、触发条件和类型
2. **确定文件结构**：
   - 结构化技能：包含可执行代码（Python/Shell等）
   - 非结构化脚本：纯Markdown描述流程
3. **创建配套文件**：
   - `README.md`：给人类看的说明文档
   - `SKILL.md`：给AI看的结构化技能说明
4. **检查完整性**：确保三个文件内容一致

## 文件结构

```
撰写技能/
├── 撰写技能.md    # 核心脚本（五要素SOP）
├── SKILL.md       # 给AI看的技能说明
└── README.md      # 给人类看的说明（本文件）
```

## 技能类型对比

| 类型 | 文件结构 | 适用场景 | 示例 |
|------|----------|----------|------|
| 结构化技能 | 包含代码文件 | 需要自动化执行 | 数据爬取、文件处理 |
| 非结构化脚本 | 纯Markdown | 流程指导、SOP | 文档修改、日志记录 |

## 配套文件说明

### README.md（给人类看）
- 技能概述（名称、版本、功能）
- 功能描述（详细说明，可使用表格）
- 使用方法（命令示例）
- 文件结构（文件列表）
- 版本历史

### SKILL.md（给AI看）
- YAML frontmatter元数据（名称、描述、emoji等）
- 触发条件
- 执行方式
- 依赖
- 输入
- 步骤
- 输出

## 注意事项

- 本脚本只输出文件内容，不涉及实际存储
- 结构化技能需要包含可执行代码
- 非结构化技能使用 Markdown 描述流程
- 三个文件内容应保持一致，只是呈现方式不同
