# 撰写技能SOP

## 触发条件
需要创建新的技能时
需要将能力封装为可复用技能时

## 依赖
无

## 输入
- 技能名称
- 技能功能描述
- 触发场景

## 步骤

### Step 1: 确认技能需求
- 明确技能命名
- 明确技能功能
- 明确触发条件
- 确定技能类型（结构化/非结构化）

### Step 2: 确定文件结构

**结构化技能**（包含代码）：
```
skills/{技能名}/
├── *.py              # Python脚本
├── *.sh              # Shell脚本
├── README.md         # 技能说明（给人类看）
└── SKILL.md          # 技能规范（给AI看）
```

**非结构化脚本**（纯Markdown）：
```
scripts/{脚本名}/
├── *.md              # 脚本内容
├── README.md         # 脚本说明（给人类看）
└── SKILL.md          # 技能规范（给AI看）
```

### Step 3: 撰写 README.md
给人类看的技能说明：
- 技能概述（名称、版本、功能）
- 功能描述（详细说明，可使用表格）
- 使用方法（命令示例）
- 文件结构（文件列表）
- 版本历史

### Step 4: 撰写 SKILL.md
给 AI 看的技能使用说明：
- 技能元数据（YAML frontmatter）
  ```yaml
  ---
  name: 技能名称
  description: >
    技能描述
  metadata:
    openclaw:
      emoji: "🔧"
      requires:
        bins: []
  ---
  ```
- 触发条件
- 执行方式
- 依赖
- 输入
- 步骤
- 输出

### Step 5: 创建 _meta.json
添加技能元数据文件：
```json
{
  "name": "技能名称",
  "description": "技能功能描述",
  "version": "1.0.0",
  "author": "大管家",
  "created_at": "YYYY-MM-DD",
  "emoji": "🔧",
  "triggers": ["触发词1", "触发词2"],
  "dependencies": {
    "bins": [],
    "python_packages": []
  },
  "environment_variables": []
}
```

### Step 6: 检查完整性
- 检查技能文件结构
- 检查三个文件的一致性

## 输出
- 核心技能文件内容（代码或Markdown）
- README.md 内容（给人类看）
- SKILL.md 内容（给AI看）
- 不涉及文件存储

## Notes
- README.md 侧重可读性，使用自然语言描述
- SKILL.md 侧重结构化，便于 AI 解析执行
- 两个文件内容应保持一致，只是呈现方式不同
- 结构化技能需要包含可执行代码
- 非结构化技能使用 Markdown 描述流程
