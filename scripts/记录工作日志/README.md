# 记录工作日志

> 版本: 1.0.0  
> 维护者: 大管家  
> 创建时间: 2026-04-08

## 功能描述

本脚本用于记录任务执行完成情况，包括任务信息、执行结果和参与Agent等详细信息。

## 使用方法

1. **确定任务类型**：如文档整理、论文写作、规则更新等
2. **填写任务描述**：简要摘要用户请求
3. **记录执行结果**：成功/失败/部分成功
4. **填写涉及项目**：项目名称或"无"
5. **记录存储路径**：生成的文件路径
6. **列出参与Agent**：如大管家、写作助手等
7. **详细说明分工**：各Agent的具体工作内容
8. **添加备注**：其他需要记录的信息
9. **保存日志文件**：按指定路径保存

## 日志文件命名规则

```
~/实验室仓库/日志文件/YYYY-MM-DD/HH-MM-SS-{AGENT}-{任务类型}.md
```

示例：
- `~/实验室仓库/日志文件/2026-04-08/14-30-00-大管家-文档整理.md`
- `~/实验室仓库/日志文件/2026-04-08/15-45-00-写作助手-论文写作.md`

## 日志内容结构

```markdown
# 工作日志

## 任务信息
- **任务类型**: [类型]
- **任务描述**: [描述]
- **执行结果**: [成功/失败/部分成功]

## 项目信息
- **涉及项目**: [项目名称]
- **存储路径**: [文件路径]

## 参与Agent
- [Agent名称]: [分工说明]

## 备注
[其他信息]
```

## 注意事项

- 任务执行完成后立即记录
- 确保文件路径准确
- 分工说明尽量详细
