### 工作记忆使用规则

> 详细规范参见 `skills/agent_self_development/working_memory/SKILL.md`
> 本文件仅记录代理特定的实践细节

#### 代理特定规则
- 为每个任务分配唯一任务ID（格式：T001, T002, ...）
- 发现偏离计划超过 10% → 暂停并更新状态为`paused`，请求用户指示

#### 定时清理
- **执行时间**：每日 00:00（Asia/Shanghai）
- **执行方式**：主代理执行
- **任务内容**：
  1. 扫描「活跃子代理清单」
  2. 将 `completed` 状态任务归档到「事件记忆」
  3. 直接删除 `killed` 状态任务（不归档）
  4. 从清单中删除已处理的任务
  5. 记录清理日志

## 程序性记忆(Procedural Memory)

### 脚本索引

| 技能名称 | 触发示例 | 描述 | 路径 |
|---------|---------|------|------|
| 学术篇章撰写脚本 | 撰写完整学术篇章 | 撰写完整学术论文章节的脚本 | `skills/学术篇章撰写脚本/SKILL.md` |
| 学术段落撰写脚本 | 撰写学术段落 | 撰写学术论文段落的脚本 | `skills/学术段落撰写脚本/SKILL.md` |
| 学术句子撰写脚本 | 撰写学术句子 | 撰写学术论文句子的脚本 | `skills/学术句子撰写脚本/SKILL.md` |

### 条件-行动规则（If-Then Rules）

| 条件 | 行动 |
|------|------|
| 需要撰写完整学术篇章 | 调用 学术篇章撰写脚本 |
| 需要撰写学术段落 | 调用 学术段落撰写脚本 |
| 需要撰写学术句子 | 调用 学术句子撰写脚本 |
| 需要并行执行多个任务 | **最多创建2个子代理**，其余任务等待或主代理执行 |
| MD文档太大（超过5000字符） | 使用 腾讯文档分段上传工具 |
| 撰写自传体记忆相关章节 | **优先使用"痕迹+图式"框架**（Fivush & Grysman, 2022）：痕迹=具体记忆内容，图式=抽象知识结构 |
| 文献综述结构选择 | **优先"从一般到特殊"递进**：一般事实记忆→自传体记忆，而非按加工阶段组织 |
| 段落中心句撰写 | **用现象描述开头**（选择性重构/视角采择/模式提取），而非"基础研究揭示了..." |
| 文献引用逻辑 | **文献必须服务论点**：先描述研究者做了什么，再用"这正是..."引出框架解释 |
| 过渡句撰写 | **过渡句是逻辑推进工具**：不仅连接段落，更要推进论证（如"这种修复不是...而是..."） |
| 章节结构选择 | **按现象/主题组织**，避免按时间顺序或加工阶段并列罗列 |
| 分析 vs 陈述区分 | **现状章节只陈述发现**，分析留给下一节，避免过早总结 |
| 章节结尾撰写 | **章节结尾必须回归核心问题**：从"现状回顾"转向"问题提出"，明确研究空白 |
| 核心概念贯穿 | **核心概念必须在全章贯穿**：在各小节间形成呼应，不能只在某一段出现 |
| 理论整合框架 | **理论整合需找到统一概念框架**：如"痕迹"作为核心概念统摄各加工阶段 |
| 小结段落撰写 | **新增小结可综合各维度**：勾勒整体转变图景，形成从具体到抽象的提升 |
| 修改计划制定 | **修改计划按优先级+章节组织**：先通读审阅意见，按P0-P6优先级和章节分布制定阶段计划 |
