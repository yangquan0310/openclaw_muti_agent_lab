### 1.3 工作记忆使用规则

#### 初始化规则（计划阶段）
- 每次复杂任务开始前，主代理必须先加载此看板，检查是否有未完成任务
- 拆解子任务后，立即添加新行到「当前活跃任务看板」，状态为`active`
- 为每个任务分配唯一任务ID（格式：T001, T002, ...）

#### 执行规则（监控阶段）
- 创建子代理后，将子代理key和分配的任务填入「活跃子代理清单」
- 每完成一个子任务 → 更新任务状态为`completed`，结果摘要填入备注
- 当需要并行执行独立子任务时 → 使用子代理，状态改为`active`
- 子代理完成（收到通告）后，将任务状态改为`completed`，结果摘要填入备注
- 发现偏离计划超过 10% → 暂停并更新状态为`paused`，请求用户指示

#### 清理规则（调节阶段）
- 任务全部结束后，将任务状态改为`completed`
- 将本次任务看板中的历史记录移动到陈述性记忆区作为归档
- 清空工作记忆区的已完成任务
- 保留`active`状态的任务用于持续监控

#### 状态说明
- `active`: 子代理正在工作或待命（执行中）
- `paused`: 任务暂停，等待用户反馈
- `completed`: 任务已完成，子代理已归档

#### 一致性维护
- 确保「当前活跃任务看板」和「活跃子代理清单」状态一致
- 每日清理已归档超过7天的历史记录
- 每次任务结束后执行闭环自检清单

## 五、程序性记忆（Procedural Memory）

### 脚本（Scripts）

#### S1 - 文献综述脚本
```
触发条件: 需要撰写文献综述
执行方式: 子代理执行
依赖: 
  - Semantic Scholar API：用于学术文献检索
  - Zotero API：用于文献管理和引用
  - S3学术段落撰写脚本：用于生成独立论证段落
  - S4学术篇章撰写脚本：用于整合段落形成完整章节
  - 通用脚本：管理文章生命流程、写入项目元数据、生成工作日志
输入: 研究主题、综述范围、文献要求
输出: 结构化文献综述文档（APA 7th格式）

步骤:
  1. 主题界定与范围确定
     - 明确综述的主题和研究领域
     - 确定文献检索的时间范围和语言范围
     - 明确纳入和排除标准

  2. 文献检索与收集
     - 使用Semantic Scholar等学术搜索引擎检索相关文献
     - 按照既定标准筛选高质量文献
     - 将检索到的文献保存到项目知识库中
     - 建立文献索引和元数据记录

  3. 文献阅读与摘要提取
     - 基于项目知识库系统阅读文献全文
     - 提取关键信息：研究目的、方法、结果、结论
     - 记录文献质量评价和操作化定义
     - 创建文献阅读笔记并存入知识库

  4. 文献分析与分类
     - 按照主题、方法、结论对文献进行分类
     - 识别研究趋势和理论脉络
     - 发现研究空白和争议点
     - 分析不同研究的操作化定义和测量方法

  5. 综述结构设计（问题导向）
     - 引言：介绍研究背景和综述重要性
     - 研究问题：梳理该领域大家关注的核心问题
     - 研究方法：分析大家如何研究这个问题，重点关注操作化定义和问卷测量方法
     - 研究结果：归纳发现的重要现象和规律
     - 讨论：解释这些现象的机制、理论贡献和研究局限
     - 未来研究方向：基于文献分析提出未来研究建议

  6. 综述撰写与整合
     - 按照问题导向结构撰写综述内容
     - 整合不同文献观点，构建理论脉络
     - 确保逻辑连贯，论证充分
     - 重点关注研究方法和操作化定义的比较分析

  7. 批判性分析与评述
     - 对现有研究的操作化定义进行批判性分析
     - 识别方法学局限和测量问题
     - 提出理论解释的替代方案
     - 总结研究范式的发展趋势

  8. 引用格式与参考文献整理
     - 采用APA 7th格式规范所有引用
     - 整理完整的参考文献列表
     - 验证引用信息的准确性和一致性
     - 确保所有引用都来自项目知识库中的真实文献

  9. 生命周期管理（调用通用脚本）
     - 调用通用脚本完成文档生命周期管理
     - 写入项目元数据（版本信息、云文档映射、修改记录）
     - 生成任务工作日志，记录综述撰写过程
```

#### S2 - 论文草稿修改脚本
```
触发条件: 需要修改和优化论文草稿
执行方式: 子代理执行
依赖: 
  - 原始论文草稿和修改要求
  - S3学术段落撰写脚本：用于重写或补充段落
  - S4学术篇章撰写脚本：用于调整篇章结构
  - 通用脚本：管理文章生命流程、更新项目元数据、生成修改日志
输入: 待修改论文草稿、修改目标、具体修改意见
输出: 优化后的论文版本

步骤:
  1. 草稿评估与问题诊断
     - 全面评估论文草稿质量
     - 识别结构、逻辑、语言等方面的问题
     - 确定修改的优先级和重点

  2. 结构与逻辑优化
     - 优化论文的整体结构
     - 增强段落间的逻辑连贯性
     - 确保论点论证的充分性

  3. 学术表达精炼
     - 提升语言的学术性和准确性
     - 简化冗余表达，增强表达效率
     - 统一专业术语和学术风格

  4. 方法论严谨性提升
     - 检查方法描述的准确性和完整性
     - 确保研究设计的科学性
     - 验证数据分析的严谨性

  5. 结果呈现优化
     - 优化结果的呈现方式
     - 确保图表和数据清晰明了
     - 增强结果的可解释性

  6. 讨论深度拓展
     - 深化研究结果的讨论
     - 增强与现有文献的对话
     - 提升理论贡献的阐述

  7. 学术规范检查
     - 检查引用格式的规范性
     - 验证参考文献的完整性
     - 确保符合目标期刊要求

  8. 修改验证与版本管理
     - 对比修改前后的变化
     - 验证修改效果
     - 建立版本管理系统

  9. 生命周期管理（调用通用脚本）
     - 调用通用脚本更新文档生命周期状态
     - 更新项目元数据（版本号、修改记录、云文档同步状态）
     - 生成修改工作日志，记录修改内容和耗时
```

#### S3 - 学术段落撰写脚本
```
触发条件: 需要撰写独立学术段落、论证特定论点、补充论文内容
执行方式: 主代理执行
依赖: 
  - 项目知识库、相关文献索引
  - 通用脚本：记录段落写作元数据、更新文档版本
输入: 论点方向、段落要求、结构偏好（总分总/总分/分总）、支撑文献范围
输出: 结构清晰、论证充分的学术段落

步骤:
  1. 论点明确与范围界定
     - 清晰界定段落核心论点，确保聚焦单一主题
     - 明确论点的适用范围和前提条件
     - 确定段落需要回答的核心研究问题

  2. 证据检索与匹配
     - 从项目知识库检索与论点相关的文献证据
     - 匹配支撑论点的实证研究结果、理论观点、权威论述
     - 评估证据的相关性、可靠性和说服力层级
     - 识别可能的反证和不同学术观点

  3. 结构选择与框架设计
     - 根据论点性质选择合适的段落结构：
       - 总分总结构：论点→论据支撑→总结升华（适用于完整独立段落）
       - 总分结构：论点→多维度论据支撑（适用于系列段落中的分论点）
       - 分总结构：多维度论据分析→归纳总结论点（适用于讨论分析类段落）
     - 设计逻辑衔接框架，确保论据间过渡自然

  4. 内容撰写与论证构建
     - 以清晰的主题句点明核心论点
     - 按照逻辑顺序呈现证据，每个证据对应论点的一个支撑维度
     - 建立论点与证据的明确联系，解释证据如何支持论点
     - 适当纳入批判性分析，体现学术严谨性
     - 按照APA 7th格式规范添加引用标注

  5. 逻辑校验与表达优化
     - 验证论点-证据链的逻辑连贯性和论证充分性
     - 优化学术表达，确保语言准确、专业、客观
     - 检查段落内部逻辑层次，避免跳跃或矛盾
     - 调整段落长度，符合学术写作规范（通常150-300字）

  6. 引用规范检查
     - 验证所有引用均来自项目知识库中的真实文献
     - 确保引用格式符合APA 7th规范
     - 整理段落涉及的参考文献条目
```

#### S4 - 学术篇章撰写脚本
```
触发条件: 需要撰写完整学术篇章（论文章节、研究报告部分、独立主题论述）
执行方式: 主代理执行
依赖: 
  - S3学术段落撰写脚本：用于生成独立论证段落
  - 项目知识库、篇章结构要求
  - 通用脚本：管理篇章生命周期、写入元数据、生成写作日志
输入: 篇章主题、核心论点、结构框架、字数要求、目标读者
输出: 逻辑连贯、结构完整的学术篇章

步骤:
  1. 篇章结构设计与论点分解
     - 明确篇章核心主题和整体论证目标
     - 设计篇章逻辑框架（通常包含引言、主体论述、结论三部分）
     - 将核心论点分解为若干层级分明的子论点
     - 确定每个子论点对应的段落数量和论证顺序

  2. 段落规划与逻辑衔接设计
     - 为每个子论点分配独立段落或段落组
     - 设计段落间的逻辑连接关系（递进、并列、因果、对比等）
     - 规划过渡句和衔接词，确保篇章整体连贯性
     - 构建论点-论据-结论的完整论证链条

  3. 批量段落撰写（调用S3脚本）
     - 针对每个子论点，调用S3学术段落撰写脚本生成独立段落
     - 统一各段落的学术风格和表达规范
     - 保持段落间的逻辑层次和论证力度平衡
     - 确保所有引用格式一致，参考文献统一管理

  4. 篇章整合与逻辑优化
     - 将独立段落按照预设结构进行整合
     - 添加段落间的过渡句和衔接词，强化逻辑关联
     - 调整段落顺序，优化论证节奏和逻辑流向
     - 检查篇章整体是否形成完整的论证闭环

  5. 内容一致性与风格统一检查
     - 验证各段落论点是否支撑篇章核心主题
     - 确保专业术语使用统一，避免前后矛盾
     - 优化学术表达风格，保持全篇一致性
     - 调整段落长度分布，符合学术写作规范

  6. 篇章质量评估与完善
     - 评估篇章的论证充分性和逻辑严谨性
     - 补充薄弱环节的论据支撑
     - 删减冗余内容，突出核心论点
     - 验证整体结构是否符合目标期刊或学术规范要求

  7. 参考文献统一整理
     - 汇总所有段落的引用文献，去重并排序
     - 按照APA 7th格式生成统一的参考文献列表
     - 验证所有引用的准确性和完整性
```
