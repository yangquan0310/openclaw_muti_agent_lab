### 1.3 工作记忆使用规则

#### 初始化规则（计划阶段）
- 每次复杂任务开始前，主代理必须先加载此看板，检查是否有未完成任务
- 拆解子任务后，立即添加新行到「当前活跃任务看板」，状态为`active`
- 为每个任务分配唯一任务ID（格式：T001, T002, ...）

#### 执行规则（监控阶段）
- 创建子代理后，将子代理key和分配的任务填入「活跃子代理清单」
- 每完成一个子任务 → 更新任务状态为`completed`，结果摘要填入备注
- 当需要并行执行独立子任务时 → 使用子代理，状态改为`active`
- 子代理完成（收到通告）后，将任务状态改为`completed`，结果摘要填入备注
- 发现偏离计划超过 10% → 暂停并更新状态为`paused`，请求用户指示


#### 清理规则（调节阶段）
- 任务全部结束后，将任务状态改为`completed`
- **只将`completed`状态的任务归档到「事件记忆」表格**（包含：日期、事件、涉及实体、结果、日志位置）
- **`killed`状态的任务直接删除，不归档**（异常终止的任务不进入历史记录）
- **从「活跃子代理清单」中删除`completed`和`killed`状态的任务**
- 只保留`active`和`paused`状态的任务在清单中
- 保留`active`状态的任务用于持续监控

#### 定时清理任务
- **执行时间**：每日凌晨04:00（Asia/Shanghai）
- **执行方式**：子代理执行
- **任务内容**：
  1. 扫描「活跃子代理清单」
  2. **只将`completed`状态的任务归档到「事件记忆」**（killed状态直接删除）
  3. 从清单中删除`completed`和`killed`状态的任务
  4. 记录清理日志（区分归档和删除的操作）


#### 状态说明
- `active`: 子代理正在工作或待命（执行中）
- `paused`: 任务暂停，等待用户反馈
- `completed`: 任务已完成，子代理已归档

#### 一致性维护
- 确保「当前活跃任务看板」和「活跃子代理清单」状态一致
- 每日清理已归档超过7天的历史记录
- 每次任务结束后执行闭环自检清单

#### S1 - 文献综述脚本

##### 触发条件
需要撰写文献综述

##### 执行方式
子代理执行

**子代理执行必须传递的完整上下文：**

- **技能**：
  - `feishu_doc`：飞书文档读写操作，用于上传文献综述到飞书云文档
  - `zotero`：Zotero文献管理，用于检索和引用文献
  - `general-scripts/管理知识库`：管理项目知识库index.json
  - `general-scripts/创建文档`：创建项目文档
  - `general-scripts/修改文档`：修改项目文档
  - `general-scripts/控制版本`：版本控制
  - `general-scripts/写入项目元数据`：写入项目元数据
  - `general-scripts/生成工作日志`：生成任务工作日志

- **密钥**：
  - `FEISHU_APP_ID` 和 `FEISHU_APP_SECRET`：飞书API访问凭证（存储于 `~/.openclaw/.env`）
  - `ZOTERO_API_KEY`：Zotero API密钥（存储于 `~/.openclaw/.env`）

- **使用方法**：
  - `feishu_doc`：使用 `feishu_doc(action="create", title="...", content="...")` 创建文档，使用 `feishu_doc(action="write", doc_token="...", content="...")` 写入内容
  - `zotero`：使用 `zotero(action="search", query="...")` 搜索文献，使用 `zotero(action="add", identifier="DOI/ISBN")` 添加文献
  - `general-scripts`：调用对应脚本文件中的SOP步骤执行

- **任务要求**：
  - 基于知识库/笔记/笔记.json中的文献笔记撰写结构化文献综述
  - 采用APA 7th格式规范所有引用
  - 输出包含引言、研究问题、研究方法、研究结果、机制解释、批判性分析与讨论、未来研究方向的完整综述
  - 确保引用文献来自笔记中的真实文献，禁止编造假文献

- **执行方案/步骤**：
  1. 读取知识库/笔记/笔记.json文件
  2. 检查文献笔记数量（≥5篇方可继续）
  3. 分析文献笔记并分类整理（实证/综述/理论）
  4. 按照综述结构撰写各部分内容
  5. 执行版本控制
  6. 根据用户需求处理云文档同步

##### 依赖
- 知识库/笔记/{笔记}_json：包含文献笔记的JSON文件
- S2学术篇章撰写脚本：用于整合段落形成完整章节
- S3学术段落撰写脚本：用于生成独立论证段落
- S4学术句子攥写脚本：用于生成句子
- 通用脚本：修改文档、管理入元数据、记录工作日志

##### 输入
- 研究主题
- 综述范围
- 笔记名称（用于生成{笔记}_json文件名）
- 指定项目知识库路径（用于定位笔记文件）
- 云文档链接（可选）

##### 步骤

1. **传递完整上下文给子代理**
   - 向子代理传递技能列表（feishu_doc、zotero、general-scripts系列）
   - 向子代理传递API密钥访问路径（`~/.openclaw/.env`）
   - 向子代理传递笔记文件路径（`~/实验室仓库/项目文件/{项目名}/知识库/笔记/{笔记}_json`）
   - 向子代理传递S4、S3、S2脚本内容作为参考
   - 明确告知子代理任务边界：仅使用笔记文件中已有文献，禁止额外检索

2. **明确子代理的任务边界**
   - 任务范围：基于笔记文件（{笔记}_json）中的文献笔记撰写文献综述
   - 禁止行为：禁止编造不存在的文献、禁止擅自进行额外文献检索
   - 质量要求：APA 7th格式、结构对仗原则、批判性分析
   - 输出规范：本地文件 + 可选云文档同步

3. **规定子代理的输出格式**
   - 文献综述正文（Markdown格式，APA 7th引用）
   - 草稿保存路径（`~/实验室仓库/项目文件/{项目名}/草稿/{标题}/{标题}_v{version}.md`）
   - 终稿保存路径（`~/实验室仓库/项目文件/{项目名}/知识库/综述/{标题}.md`）
   - 笔记文件保存路径（`~/实验室仓库/项目文件/{项目名}/知识库/笔记/{笔记}_json`）
   - 云文档链接（如适用）
   - 执行摘要（包含文献统计、主要发现、研究空白）

4. **读取笔记文件并检查**
   - 子代理打开笔记文件（`~/实验室仓库/项目文件/{项目名}/知识库/笔记/{笔记}_json`）
   - 解析JSON结构，获取文献笔记列表
   - 统计文献笔记数量
   - 若文献笔记<5篇 → 直接返回："笔记文献不足，请先补充笔记后再撰写综述"

   - 明确综述的主题和研究领域

   - 明确纳入和排除标准（仅使用笔记.json中已有文献）

6. **分类与分析文献笔记**
   - 按文献类型分类：实证类（研究问题、方法、结果、结论）、综述类（问题、结果、展望）、理论类（问题、观点）
   - 按时间、作者对文献分类
   - 汇总文献研究的问题
   - 分析现有研究采用的方法（实证类），重点关注实验法和问卷法的操作性定义
   - 汇总发现的效应（实证类）
   - 汇总效应的解释（实证类）
   - 汇总观点和理论框架（理论类）
   - 汇总综述结果和展望（综述类）
   - 批判性分析：对文献分类结果进行初步批判性分析，识别研究空白和方法局限

7. **撰写综述文章**
   
   - **综述结构**：
     - 引言：介绍研究背景和综述重要性
     - 研究问题：梳理该领域关注的核心问题
     - 研究方法：分析现有研究采用的方法，重点关注操作化定义
     - 研究结果：归纳发现的重要现象和规律
     - 机制解释：解释这些现象的机制
     - 批判性分析与讨论：对现有研究进行批判性分析，讨论理论贡献和研究局限
     - 未来研究方向：基于文献分析提出未来研究建议
   
   - **段落要求**：
     - 论点明确
     - 证据匹配
   
   - **篇章要求**：
     - **结构对仗原则**：
       - 研究结果章节应与研究问题章节对仗，按问题组织文献发现，而非按文献分类
       - 机制解释章节应与研究结果章节对仗，解释每个研究发现背后的原因
     - 添加段落间的过渡句和衔接词
     - 调整段落顺序，优化论证节奏
     - 验证各段落论点是否支撑综述核心主题
     - 确保专业术语使用统一
     - 确保逻辑连贯，论证充分
   
   - **格式要求**：
     - 采用APA 7th格式规范所有引用
     - 整理完整的参考文献列表
     - 验证引用信息与知识库元数据完全一致

10. **主代理接收、验证与更新元数据**
    - 接收子代理返回的文献综述正文、文件路径、云文档链接
    - 验证本地文件是否存在且内容完整
    - 验证引用格式是否符合APA 7th规范
    - **主代理更新项目元数据**（子代理不执行）
    - 验证工作日志是否正确生成
    - 如有问题，要求子代理修正

- 本地文件路径
- 云文档链接（如用户确认上传或已提供链接）

   - 验证整体结构是否符合目标期刊或学术规范要求

9. **整理参考文献**

    - 按照APA 7th格式生成统一的参考文献列表
    - 验证所有引用的准确性和完整性

#### S3 - 学术段落撰写脚本

##### 触发条件
需要撰写独立学术段落、论证特定论点、补充论文内容

##### 执行方式
当前会话执行（主代理在计划阶段调用，作为路由指引）

**路由指引：**
- 主代理在计划撰写段落时调用S3
- S3提供段落撰写的标准流程、结构选择和规范要求
- 主代理根据S3的指导：
  - 若段落复杂或需要大量文献支撑 → 创建子代理执行段落撰写
  - 若段落简单且主代理可直接完成 → 按照S3规范自行撰写段落
- S3是S2（学术篇章）和S1（文献综述）的底层组件

##### 依赖
- 项目知识库、相关文献索引
- S4 撰写学术句子（提供句子级规范）

##### 输入
- 论点方向（段落要论证的核心观点）
- 段落要求（如：论证深度、学术风格、目标读者）
- 结构偏好（总分总/总分/分总）
- 支撑文献范围（指定知识库中的相关文献）

##### 步骤

1. **分析论点与确定范围**
   - 清晰界定段落核心论点，确保聚焦单一主题
   - 明确论点的适用范围和前提条件
   - 确定段落需要回答的核心研究问题

2. **检索与评估证据**
   - 从项目知识库检索与论点相关的文献证据
   - 匹配支撑论点的实证研究结果、理论观点、权威论述
   - 评估证据的相关性、可靠性和说服力层级
   - 识别可能的反证和不同学术观点

3. **选择段落结构**
   - 根据论点性质选择合适的段落结构：
     - **总分总结构**：论点→论据支撑→总结升华（适用于完整独立段落）
     - **总分结构**：论点→多维度论据支撑（适用于系列段落中的分论点）
     - **分总结构**：多维度论据分析→归纳总结论点（适用于讨论分析类段落）
   - 设计逻辑衔接框架，确保论据间过渡自然

4. **撰写段落（调用S4规范）**
   - 以清晰的主题句点明核心论点
   - 按照逻辑顺序呈现证据，每个证据对应论点的一个支撑维度
   - 建立论点与证据的明确联系，解释证据如何支持论点
   - 适当纳入批判性分析，体现学术严谨性
   - 按照APA 7th格式规范添加引用标注
   - **调用S4规范**：确保句子符合实体主语、动词行动链等要求

5. **校验与优化**
   - 验证论点-证据链的逻辑连贯性和论证充分性
   - 优化学术表达，确保语言准确、专业、客观
   - 检查段落内部逻辑层次，避免跳跃或矛盾
   - 调整段落长度，符合学术写作规范（通常150-300字）

6. **检查引用规范**
   - 验证所有引用均来自项目知识库中的真实文献
   - 确保引用格式符合APA 7th规范
   - 整理段落涉及的参考文献条目

##### 输出
- 结构清晰、论证充分的学术段落（Markdown格式）
- 段落结构说明（总分总/总分/分总）
- 引用文献列表

#### S4 - 撰写句子脚本

##### 触发条件
需要撰写符合学术规范的中英文句子

##### 执行方式
当前会话执行（主代理在计划阶段调用，作为基础规范）

**路由指引：**
- 主代理在撰写任何学术文本时调用S4
- S4提供句子撰写的核心规范（实体主语、动词行动链、概念定义格式）
- 主代理根据S4的指导：
  - 检查句子是否符合规范
  - 修正不符合规范的句子
  - 作为S3（学术段落）和S2（学术篇章）的基础组件
- S4是最底层的写作规范脚本

##### 依赖
- 无（基础规范脚本）

##### 输入
- 句子主题或核心概念
- 语言类型（中文/英文）
- 句子类型（陈述/定义/论证/描述）
- 相关实体名称

##### 步骤

1. **确定句子主语**
   - 识别核心实体名称
   - 确保主语明确具体
   - 避免使用抽象概念作为主语
   - **规范**：以实体名称作为主语

2. **构建动词行动链**
   - 选择准确的动词描述主体行动
   - 确保动作链条清晰连贯
   - 中文可使用把字句或被动句优化表达
   - **规范**：使用动词作为行动链，强调主体的行动

3. **处理概念定义（如适用）**
   - 采用"是...的..."结构
   - 使用定语修饰核心名词
   - **规范**：概念定义格式 = "是" + 定语 + 定语 + "的" + 名词
   - 示例："自传体记忆是个体对过去个人经历的有意识回忆"

4. **优化中文表达**
   - 检查是否以实体为主语
   - 验证动词行动链是否清晰
   - 避免不必要的抽象名词（慎用抽象名词，除非特定概念）
   - 确保句子通顺自然

5. **转换英文表达（如需要）**
   - 遵循英文学术写作规范
   - 调整语序符合英文习惯
   - 使用恰当的学术词汇
   - 确保语法正确

6. **验证句子质量**
   - 检查主语是否明确
   - 验证动词使用是否准确
   - 确保符合目标语言规范
   - 确认表达简洁清晰

##### 输出
- 符合规范的中文或英文句子
- 句子结构分析（主语、谓语、宾语/补语）
- 概念定义格式验证（如适用）
- 规范检查清单（实体主语✓、动词行动链✓、无抽象名词✓）
