### 1.3 工作记忆使用规则

#### 初始化规则（计划阶段）
- 每次复杂任务开始前，主代理必须先加载此看板，检查是否有未完成任务
- 拆解子任务后，立即添加新行到「当前活跃任务看板」，状态为`active`
- 为每个任务分配唯一任务ID（格式：T001, T002, ...）

#### 执行规则（监控阶段）
- 创建子代理后，将子代理key和分配的任务填入「活跃子代理清单」
- 每完成一个子任务 → 更新任务状态为`completed`，结果摘要填入备注
- 当需要并行执行独立子任务时 → 使用子代理，状态改为`active`
- 子代理完成（收到通告）后，将任务状态改为`completed`，结果摘要填入备注
- 发现偏离计划超过 10% → 暂停并更新状态为`paused`，请求用户指示

#### 清理规则（调节阶段）
- 任务全部结束后，将任务状态改为`completed`
- 将本次任务看板中的历史记录移动到陈述性记忆区作为归档
- 清空工作记忆区的已完成任务
- 保留`active`状态的任务用于持续监控

#### 状态说明
- `active`: 子代理正在工作或待命（执行中）
- `paused`: 任务暂停，等待用户反馈
- `completed`: 任务已完成，子代理已归档

#### 一致性维护
- 确保「当前活跃任务看板」和「活跃子代理清单」状态一致
- 每日清理已归档超过7天的历史记录
- 每次任务结束后执行闭环自检清单

#### S1 - 文献综述脚本

触发条件: 需要撰写文献综述

执行方式: 主代理执行（子代理可选）

依赖: 
  - 项目知识库：优先使用已有知识库文献，无需额外检索
  - S3学术段落撰写脚本：用于生成独立论证段落
  - S4学术篇章撰写脚本：用于整合段落形成完整章节
  - 通用脚本：创建文档、修改文档、版本控制、写入元数据、生成工作日志

输入: 
  - 研究主题
  - 综述范围
  - 指定项目知识库路径
  - 云文档链接（可选）

步骤:
  1. 知识库检查
     - 打开指定项目知识库的index.json
     - 统计与主题相关的文献数量
     - 若相关文献<5篇 → 直接返回："知识库文献不足，请先补充文献后再撰写综述"
     - 若相关文献≥5篇 → 继续执行

  2. 主题界定与范围确定
     - 明确综述的主题和研究领域
     - 基于知识库文献确定时间范围和内容范围
     - 明确纳入和排除标准（仅使用知识库中已有文献）

  3. 文献分类与分析
     - 按时间、作者、杂志对文献分类
     - 汇总文献研究的问题
     - 分析现有研究采用的方法，重点关注实验法和问卷法的操作性定义
     - 汇总发现的效应
     - 汇总效应的解释
     - 批判性分析：对文献分类结果进行初步批判性分析，识别研究空白和方法局限

  4. 撰写文章
     
     - 综述结构：
       - 引言：介绍研究背景和综述重要性
       - 研究问题：梳理该领域关注的核心问题
       - 研究方法：分析现有研究采用的方法，重点关注操作化定义
       - 研究结果：归纳发现的重要现象和规律
       - 机制解释：解释这些现象的机制
       - 批判性分析与讨论：对现有研究进行批判性分析，讨论理论贡献和研究局限
       - 未来研究方向：基于文献分析提出未来研究建议
     
     - 段落要求：
       - 论点明确
       - 证据匹配
     
     - 篇章要求：
       - **结构对仗原则**：
         - 研究结果章节应与研究问题章节对仗，按问题组织文献发现，而非按文献分类
         - 机制解释章节应与研究结果章节对仗，解释每个研究发现背后的原因
       - 添加段落间的过渡句和衔接词
       - 调整段落顺序，优化论证节奏
       - 验证各段落论点是否支撑综述核心主题
       - 确保专业术语使用统一
       - 确保逻辑连贯，论证充分
     
     - 格式要求：
       - 采用APA 7th格式规范所有引用
       - 整理完整的参考文献列表
       - 验证引用信息与知识库元数据完全一致

  5. 版本控制
     - 草稿管理：将撰写过程中的版本保存到草稿目录
     - 终稿管理：完成终稿后保存到终稿目录
     - 备份旧版本：创建备份目录 `草稿/{filename}/`，复制当前文件到备份目录，命名格式：`{filename}_v{old_version}.md`
     - 更新当前文件：终稿目录中的当前文件命名保持为 `{filename}.md`
     - 写入项目元数据（版本信息、云文档映射、修改记录）
     - 生成任务工作日志，记录综述撰写过程

  6. 云文档处理
     - 检查输入中是否提供云文档链接
     - 若未提供云文档链接 → 询问用户："是否需要上传文献综述到云文档？"
       - 用户确认上传 → 调用通用脚本上传至云文档，更新项目元数据
       - 用户拒绝上传 → 仅保存本地文件，记录到工作日志
     - 若提供云文档链接 → 调用通用脚本同步更新到指定云文档

输出: 
  - 结构化文献综述文档（APA 7th格式）
  - 本地文件路径
  - 云文档链接（如用户确认上传或已提供链接）

#### S2 - 论文草稿修改脚本
```
触发条件: 需要修改和优化论文草稿
执行方式: 子代理执行
依赖: 
  - 原始论文草稿和修改要求
  - S3学术段落撰写脚本：用于重写或补充段落
  - S4学术篇章撰写脚本：用于调整篇章结构
  - 通用脚本：管理文章生命流程、更新项目元数据、生成修改日志
输入: 待修改论文草稿、修改目标、具体修改意见
输出: 优化后的论文版本

步骤:
  1. 草稿评估与问题诊断
     - 全面评估论文草稿质量
     - 识别结构、逻辑、语言等方面的问题
     - 确定修改的优先级和重点

  2. 结构与逻辑优化
     - 优化论文的整体结构
     - 增强段落间的逻辑连贯性
     - 确保论点论证的充分性

  3. 学术表达精炼
     - 提升语言的学术性和准确性
     - 简化冗余表达，增强表达效率
     - 统一专业术语和学术风格

  4. 方法论严谨性提升
     - 检查方法描述的准确性和完整性
     - 确保研究设计的科学性
     - 验证数据分析的严谨性

  5. 结果呈现优化
     - 优化结果的呈现方式
     - 确保图表和数据清晰明了
     - 增强结果的可解释性

  6. 讨论深度拓展
     - 深化研究结果的讨论
     - 增强与现有文献的对话
     - 提升理论贡献的阐述

  7. 学术规范检查
     - 检查引用格式的规范性
     - 验证参考文献的完整性
     - 确保符合目标期刊要求

  8. 修改验证与版本管理
     - 对比修改前后的变化
     - 验证修改效果
     - 建立版本管理系统

  9. 生命周期管理（调用通用脚本）
     - 调用通用脚本更新文档生命周期状态
     - 更新项目元数据（版本号、修改记录、云文档同步状态）
     - 生成修改工作日志，记录修改内容和耗时
```

#### S3 - 学术段落撰写脚本
```
触发条件: 需要撰写独立学术段落、论证特定论点、补充论文内容
执行方式: 主代理执行
依赖: 
  - 项目知识库、相关文献索引
  - 通用脚本：记录段落写作元数据、更新文档版本
输入: 论点方向、段落要求、结构偏好（总分总/总分/分总）、支撑文献范围
输出: 结构清晰、论证充分的学术段落

步骤:
  1. 论点明确与范围界定
     - 清晰界定段落核心论点，确保聚焦单一主题
     - 明确论点的适用范围和前提条件
     - 确定段落需要回答的核心研究问题

  2. 证据检索与匹配
     - 从项目知识库检索与论点相关的文献证据
     - 匹配支撑论点的实证研究结果、理论观点、权威论述
     - 评估证据的相关性、可靠性和说服力层级
     - 识别可能的反证和不同学术观点

  3. 结构选择与框架设计
     - 根据论点性质选择合适的段落结构：
       - 总分总结构：论点→论据支撑→总结升华（适用于完整独立段落）
       - 总分结构：论点→多维度论据支撑（适用于系列段落中的分论点）
       - 分总结构：多维度论据分析→归纳总结论点（适用于讨论分析类段落）
     - 设计逻辑衔接框架，确保论据间过渡自然

  4. 内容撰写与论证构建
     - 以清晰的主题句点明核心论点
     - 按照逻辑顺序呈现证据，每个证据对应论点的一个支撑维度
     - 建立论点与证据的明确联系，解释证据如何支持论点
     - 适当纳入批判性分析，体现学术严谨性
     - 按照APA 7th格式规范添加引用标注

  5. 逻辑校验与表达优化
     - 验证论点-证据链的逻辑连贯性和论证充分性
     - 优化学术表达，确保语言准确、专业、客观
     - 检查段落内部逻辑层次，避免跳跃或矛盾
     - 调整段落长度，符合学术写作规范（通常150-300字）

  6. 引用规范检查
     - 验证所有引用均来自项目知识库中的真实文献
     - 确保引用格式符合APA 7th规范
     - 整理段落涉及的参考文献条目
```

#### S4 - 学术篇章撰写脚本
```
触发条件: 需要撰写完整学术篇章（论文章节、研究报告部分、独立主题论述）
执行方式: 主代理执行
依赖: 
  - S3学术段落撰写脚本：用于生成独立论证段落
  - 项目知识库、篇章结构要求
  - 通用脚本：管理篇章生命周期、写入元数据、生成写作日志
输入: 篇章主题、核心论点、结构框架、字数要求、目标读者
输出: 逻辑连贯、结构完整的学术篇章

步骤:
  1. 篇章结构设计与论点分解
     - 明确篇章核心主题和整体论证目标
     - 设计篇章逻辑框架（通常包含引言、主体论述、结论三部分）
     - 将核心论点分解为若干层级分明的子论点
     - 确定每个子论点对应的段落数量和论证顺序

  2. 段落规划与逻辑衔接设计
     - 为每个子论点分配独立段落或段落组
     - 设计段落间的逻辑连接关系（递进、并列、因果、对比等）
     - 规划过渡句和衔接词，确保篇章整体连贯性
     - 构建论点-论据-结论的完整论证链条

  3. 批量段落撰写（调用S3脚本）
     - 针对每个子论点，调用S3学术段落撰写脚本生成独立段落
     - 统一各段落的学术风格和表达规范
     - 保持段落间的逻辑层次和论证力度平衡
     - 确保所有引用格式一致，参考文献统一管理

  4. 篇章整合与逻辑优化
     - 将独立段落按照预设结构进行整合
     - 添加段落间的过渡句和衔接词，强化逻辑关联
     - 调整段落顺序，优化论证节奏和逻辑流向
     - 检查篇章整体是否形成完整的论证闭环

  5. 内容一致性与风格统一检查
     - 验证各段落论点是否支撑篇章核心主题
     - 确保专业术语使用统一，避免前后矛盾
     - 优化学术表达风格，保持全篇一致性
     - 调整段落长度分布，符合学术写作规范

  6. 篇章质量评估与完善
     - 评估篇章的论证充分性和逻辑严谨性
     - 补充薄弱环节的论据支撑
     - 删减冗余内容，突出核心论点
     - 验证整体结构是否符合目标期刊或学术规范要求

  7. 参考文献统一整理
     - 汇总所有段落的引用文献，去重并排序
     - 按照APA 7th格式生成统一的参考文献列表
     - 验证所有引用的准确性和完整性
```
