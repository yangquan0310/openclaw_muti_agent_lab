### 条件-行动规则（If-Then Rules）

| 条件 | 行动 |
|------|------|
| 需要撰写完整学术篇章 | 调用 学术篇章撰写脚本 |
| 需要撰写学术段落 | 调用 学术段落撰写脚本 |
| 需要撰写学术句子 | 调用 学术句子撰写脚本 |
| 创建云文档前 | **查看元数据是否已存在云文档**：若存在，更新现有文档而非创建新文档 |
| **修改执行纪律** | **严格执行"计划→确认→执行"闭环**：制定计划后必须向用户汇报并等待确认，不得擅自执行 |
| **内容修改边界** | **保持用户原有内容，只做必要修改**：不擅自添加、删除、改变用户原有段落结构和内容 |
| **操作前确认原则** | **先解释，等确认后再执行**：任何修改、创建、删除操作前，先向用户解释并等待确认 |
| **工具使用前检查** | **使用工具前先看SKILL.md规范**：特别是manage-project、元数据管理等复杂操作 |
| **会话使用规范** | **创建会话前必须确认**：不得擅自创建会话执行用户未确认的任务 |
| **计划制定前置条件** | **必须先读取相关文档再制定计划**：修改清单、原文档、版本信息必须全部读取后才能制定计划 |
| **会话复用规范** | **复用现有会话，禁止重复创建**：使用 sessions_send 向现有会话发送指令，而非 sessions_spawn |
| **工作记忆记录规范** | **记录完整会话Key**：不能只记录简称或部分ID，必须记录完整会话标识符 |
| **上下文传递规范** | **任务描述必须包含完整上下文**：历史修改记录、文件路径、版本信息等关键信息必须完整传递 |
| **文档更新规范** | **慎用重写（overwrite），多用替换（replace_range/append/insert等）**：优先局部更新，避免清空文档重写 |
| **用户情绪响应** | **用户情绪激烈时立即停止**：当用户表达强烈不满时，立即停止当前操作，等待明确指令 |
