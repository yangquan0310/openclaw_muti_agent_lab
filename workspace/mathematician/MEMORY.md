### 1.3 工作记忆使用规则

工作记忆使用规则

##### 初始化规则（计划阶段）
- 每次复杂任务开始前，主代理必须先加载此看板，检查是否有未完成任务
- 拆解子任务后，立即添加新行到「当前活跃任务看板」，状态为`active`
- 为每个任务分配唯一任务ID（格式：T001, T002, ...）

##### 执行规则（监控阶段）
- 创建子代理后，将子代理key和分配的任务填入「活跃子代理清单」
- 每完成一个子任务 → 更新任务状态为`completed`，结果摘要填入备注
- 当需要并行执行独立子任务时 → 使用子代理，状态改为`active`
- 子代理完成（收到通告）后，将任务状态改为`completed`，结果摘要填入备注
- 发现偏离计划超过 10% → 暂停并更新状态为`paused`，请求用户指示

##### 清理规则（调节阶段）
- 任务全部结束后，将任务状态改为`completed`
- **只将`completed`状态的任务归档到「事件记忆」表格**（包含：日期、事件、涉及实体、结果、日志位置）
- **`killed`状态的任务直接删除，不归档**（异常终止的任务不进入历史记录）
- **从「活跃子代理清单」中删除`completed`和`killed`状态的任务**
- 只保留`active`和`paused`状态的任务在清单中
- 保留`active`状态的任务用于持续监控

##### 定时清理任务
- **执行时间**：每日凌晨04:00（Asia/Shanghai）
- **执行方式**：子代理执行
- **任务内容**：
  1. 扫描「活跃子代理清单」
  2. **只将`completed`状态的任务归档到「事件记忆」**（killed状态直接删除）
  3. 从清单中删除`completed`和`killed`状态的任务
  4. 记录清理日志（区分归档和删除的操作）

##### 状态说明与处理规则

| 状态 | 处理方式 | 说明 |
|------|----------|------|
| `active` | **保留在清单中** | 子代理正在工作或待命（执行中） |
| `paused` | **保留在清单中** | 任务暂停，等待用户反馈 |
| `completed` | **归档到事件记忆后删除** | 任务正常完成，归档到历史记录 |
| `killed` | **直接删除，不归档** | 任务异常终止，不进入历史记录 |

##### 一致性维护
- 确保「当前活跃任务看板」和「活跃子代理清单」状态一致
- **每日凌晨04:00自动清理**：`completed`任务归档到事件记忆，`killed`任务直接删除
- 每次任务结束后执行闭环自检清单
## 二、陈述性记忆（Semantic Memory）

### 知识网络

#### 常用统计方法
| 方法名称 | 适用场景 | 前提假设 | 输出结果 |
|----------|----------|----------|----------|
| t检验 | 两组均值比较 | 正态性、方差齐性 | t值、p值、置信区间 |
| 卡方检验 | 分类变量关联 | 期望频数>5 | χ²值、p值、效应量 |
| 相关分析 | 变量间关系 | 线性关系、正态性 | r值、p值、决定系数 |
| 方差分析 | 多组比较 | 正态性、方差齐性、独立性 | F值、p值、效应量 |
| 回归分析 | 预测建模 | 线性关系、残差正态性 | 回归系数、R²、p值 |

#### 方法选择决策树
```
数据类型
├── 连续变量
│   ├── 两组比较 → t检验 / Mann-Whitney U
│   ├── 多组比较 → ANOVA / Kruskal-Wallis
│   ├── 相关分析 → Pearson / Spearman相关
│   └── 预测建模 → 回归分析 / 机器学习
│
└── 分类变量
    ├── 两组比较 → 卡方检验 / Fisher精确检验
    ├── 关联分析 → 卡方检验 / 列联系数
    └── 预测建模 → 逻辑回归 / 分类算法
```

## 五、程序性记忆（Procedural Memory）

### 脚本（Scripts）

#### S1：统计咨询
```
触发条件: 用户需要统计方法建议或实验设计分析
执行方式: 创建子代理执行
依赖: 统计专业知识库、方法选择决策树
输入: 研究问题、数据类型、样本量、研究设计
输出: 推荐统计方法、前提假设、注意事项、实施步骤

步骤:
  1. 分析研究问题类型
     - 识别研究目标：描述、比较、关联、预测
     - 确定变量类型：连续/分类/有序
     - 确认研究设计：横断面/纵向/实验

  2. 选择统计方法
     - 参考方法选择决策树
     - 考虑样本量大小
     - 检查前提假设条件

  3. 提供方法建议
     - 推荐主选方法及理由
     - 提供备选方案
     - 说明前提假设和检验方法

  4. 设计实施步骤
     - 数据预处理步骤
     - 分析方法执行流程
     - 结果解释指南

  5. 输出建议报告
     - 包含方法选择理由
     - 前提假设检查清单
     - 实施步骤详细说明
```

#### S2：数据分析
```
触发条件: 需要执行统计分析和建模
执行方式: 创建子代理执行
依赖: 统计软件/工具、数据预处理脚本
输入: 原始数据、分析目标、变量信息
输出: 分析结果、可视化图表、统计报告

步骤:
  1. 数据预处理
     - 缺失值处理
     - 异常值检测
     - 变量转换（如需要）
     - 数据标准化

  2. 探索性数据分析
     - 描述性统计
     - 数据分布检查
     - 相关性分析
     - 数据可视化

  3. 执行统计分析
     - 根据研究目标选择方法
     - 运行统计模型
     - 检查模型假设
     - 调整模型参数

  4. 结果解释
     - 提取统计显著性
     - 计算效应量
     - 解释实际意义

  5. 生成分析报告
     - 包含主要发现
     - 可视化结果
     - 方法局限性说明
```

#### S3：方法验证
```
触发条件: 需要验证统计方法适用性或检查假设条件
执行方式: 创建子代理执行
依赖: 假设检验方法、诊断工具
输入: 数据、统计方法、假设条件
输出: 假设检验结果、诊断报告、方法建议

步骤:
  1. 识别假设条件
     - 正态性检验
     - 方差齐性检验
     - 独立性检验
     - 线性关系检验

  2. 执行诊断检验
     - Shapiro-Wilk检验（正态性）
     - Levene检验（方差齐性）
     - Durbin-Watson检验（独立性）
     - 残差分析（线性）

  3. 评估假设满足程度
     - 判断是否满足假设
     - 评估违反程度
     - 分析可能影响

  4. 提供解决方案
     - 数据转换建议
     - 替代方法推荐
     - 稳健方法选择

  5. 生成验证报告
     - 包含诊断结果
     - 假设满足程度评估
     - 改进建议
```

#### S4：数学建模
```
触发条件: 需要建立数学模型或推导数学公式
执行方式: 创建子代理执行
依赖: 数学理论、建模工具
输入: 实际问题、已知条件、约束条件
输出: 数学模型、公式推导、参数估计、模型验证

步骤:
  1. 问题抽象化
     - 识别关键变量
     - 建立变量间关系
     - 确定约束条件

  2. 构建数学模型
     - 选择适当数学形式
     - 建立方程/不等式
     - 定义参数和变量

  3. 模型求解
     - 解析求解（如可能）
     - 数值求解（如需要）
     - 参数估计

  4. 模型验证
     - 检查模型合理性
     - 验证预测能力
     - 敏感性分析

  5. 生成建模报告
     - 包含模型描述
     - 推导过程
     - 验证结果
     - 应用建议
```

### 条件-行动规则（If-Then Rules）

| 条件 | 行动 |
|------|------|
| 用户询问统计方法选择 | 调用 S1 统计咨询 |
| 需要执行数据分析 | 调用 S2 数据分析 |
| 需要验证统计方法适用性 | 调用 S3 方法验证 |
| 需要建立数学模型 | 调用 S4 数学建模 |
| 数据不满足正态性假设 | 建议非参数方法或数据转换 |
| 样本量过小 | 建议使用精确检验或增加样本量 |
| 存在多重共线性 | 建议主成分分析或岭回归 |
| 时间序列数据 | 建议时间序列分析方法 |
| 分类预测问题 | 建议逻辑回归或分类算法 |
| 需要处理缺失数据 | 建议多重插补或模型基础方法 |
