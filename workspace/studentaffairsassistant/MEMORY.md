### 工作记忆使用规则

#### 初始化规则（计划阶段）
- 每次复杂任务开始前，主代理必须先加载此看板，检查是否有未完成任务
- 拆解子任务后，立即添加新行到「当前活跃任务看板」，状态为`active`
- 为每个任务分配唯一任务ID（格式：T001, T002, ...）

#### 执行规则（监控阶段）
- 创建子代理后，将子代理key和分配的任务填入「活跃子代理清单」
- 每完成一个子任务 → 更新任务状态为`completed`，结果摘要填入备注
- 当需要并行执行独立子任务时 → 使用子代理，状态改为`active`
- 子代理完成（收到通告）后，将任务状态改为`completed`，结果摘要填入备注
- 发现偏离计划超过 10% → 暂停并更新状态为`paused`，请求用户指示

#### 清理规则（调节阶段）
- 任务全部结束后，将任务状态改为`completed`
- 将本次任务看板中的历史记录移动到陈述性记忆区作为归档
- 清空工作记忆区的已完成任务
- 保留`active`状态的任务用于持续监控

#### 状态说明
- `active`: 子代理正在工作或待命（执行中）
- `paused`: 任务暂停，等待用户反馈
- `completed`: 任务已完成，子代理已归档

#### 一致性维护
- 确保「当前活跃任务看板」和「活跃子代理清单」状态一致
- 每日清理已归档超过7天的历史记录
- 每次任务结束后执行闭环自检清单

### 脚本（Scripts）

#### 脚本1：学生信息管理

**触发条件**
- 用户提及学生信息、档案、资料管理需求
- 需要创建、更新或查询学生信息

**执行方式**
- 主代理直接执行（简单查询/记录）
- 复杂任务创建子代理

**依赖**
- 学生信息管理规范
- 数据隐私保护要求

**输入**
- 学生基本信息（姓名、学号、班级等）
- 相关附件或文档
- 操作指令（创建、查询、更新、删除）

**输出**
- 学生档案存储路径
- 操作确认信息
- 查询结果汇总

**步骤**
```
1. 识别需求类型：创建、查询、更新或删除
2. 检查输入完整性
3. 验证数据格式和规范
4. 选择存储位置：~/教研室仓库/学生工作/学生档案/
5. 执行相应操作：
   - 创建：建立档案文件夹，存储基本信息
   - 查询：检索档案，返回相关信息
   - 更新：修改档案内容，记录变更历史
   - 删除：标记档案状态，保留历史记录
6. 记录操作日志
7. 返回操作结果和存储路径
```

#### 脚本2：学业辅导

**触发条件**
- 用户提及学业问题、学习困难、成绩提升
- 需要学业辅导或学习建议

**执行方式**
- 主代理直接执行（简单建议）
- 复杂问题创建子代理

**依赖**
- 学业辅导流程规范
- 学习资源库

**输入**
- 学生学业情况描述
- 具体问题或困难
- 期望目标

**输出**
- 学业辅导方案
- 学习资源推荐
- 跟进计划

**步骤**
```
1. 分析学业问题类型：课程困难、学习方法、时间管理等
2. 评估问题严重程度和紧迫性
3. 制定个性化辅导方案：
   - 短期应对策略
   - 长期学习规划
   - 资源推荐（教材、辅导资料、学习工具）
4. 协调辅导资源：
   - 教师辅导时间安排
   - 学习小组组建建议
   - 在线学习资源链接
5. 制定跟进计划和时间表
6. 记录辅导过程和结果
7. 定期评估效果，调整方案
```

#### 脚本3：社团活动记录

**触发条件**
- 用户提及社团活动、俱乐部管理、学生组织
- 需要记录或组织社团活动

**执行方式**
- 主代理直接执行（简单记录）
- 复杂活动创建子代理

**依赖**
- 社团管理规范
- 活动策划流程

**输入**
- 社团活动基本信息（名称、时间、地点、参与人员）
- 活动计划和目标
- 所需资源和预算

**输出**
- 活动计划文档
- 活动记录总结
- 资源协调结果

**步骤**
```
1. 收集活动基本信息
2. 制定详细活动计划：
   - 时间安排和流程
   - 人员分工
   - 物资准备
   - 场地安排
3. 协调资源和审批：
   - 场地申请
   - 物资采购或借用
   - 经费预算
4. 活动实施记录：
   - 参与人员签到
   - 活动过程记录（文字、照片、视频）
   - 现场问题处理
5. 活动后总结：
   - 效果评估
   - 问题分析和改进建议
   - 成果展示
6. 归档活动资料
7. 生成活动报告
```
