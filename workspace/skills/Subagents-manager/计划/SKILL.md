# 计划阶段

## 概述

计划阶段是「计划 → 监控 → 调节」闭环的第一阶段，负责分析用户需求，制定详细的执行计划。

## 文件说明

| 文件 | 功能 | 说明 |
|------|------|------|
| `SKILL.md` | 阶段规范 | 计划阶段详细执行规范 |

## 工作流

### 1. 需求分析与原则确认

- 解析用户需求
- 不懂就问

### 2. 目标与约束梳理

- 用1句话复述用户的核心目标
- 列出关键约束：时间、权限、资源、安全边界

### 3. 任务拆解与子代理规划

- 将目标拆解为可执行的子任务，标注依赖关系
- 检查是否有可复用的活跃子代理，优先复用
- 规划子代理创建方案，确定子代理角色和任务分配

### 4. 风险与成功标准定义

- 预判可能的卡点或需要用户提供的信息
- 定义可验证的完成条件

### 5. 整合与汇报

- 整合上述内容形成完整计划方案
- 按照输出格式整理为用户可见的执行计划
```markdown
📋 **执行计划**
- 目标：…
- 子任务：① … → ② … → ③ …
- 执行方式：默认使用子代理执行
- 任务看板预览：（可选）
- 成功标准：…
- **计划已制定完成，请确认是否可以开始执行？**
```

---

## 使用指南

### 调用方式

用户发布复杂任务后，由主代理自动遵循，无需手动调用。
