# 调节阶段

## 概述

调节阶段是「计划 → 监控 → 调节」闭环的第三阶段，负责任务收尾、经验总结和知识沉淀。

## 文件说明

| 文件 | 功能 | 说明 |
|------|------|------|
| `SKILL.md` | 阶段规范 | 调节阶段详细执行规范 |

## 工作流



### 1. 差异与经验总结

- 总结实际执行与计划的差异及原因
- 沉淀经验到记忆系统：
  - **核心事实** → 写入 MEMORY.md 的「陈述性记忆区」
  - **问题及解决方案** → 写入 MEMORY.md 的「程序性记忆区」的「条件-行动」规则
  - **成功的任务拆分/子代理使用模式** → 考虑固化为SOP，写入「脚本」区域
  - **流程缺陷** → 更新条件-行动规则，避免重复问题

### 2. 任务日志撰写

- 按规范撰写完整任务日志
- 日志存放位置：`~/实验室仓库/日志文件/YYYY-MM-DD/HH-MM-SS-[agents]-[任务类型].md`
- 日志包含字段：
  - 时间戳
  - 任务ID
  - 任务目标
  - 涉及项目
  - 最终结果
  - 关键步骤摘要
  - 耗时
  - 问题与解决方案
  - 核查结果

### 3. 工作记忆清理与归档

- 将本次任务看板中的历史记录移动到 MEMORY.md 的「陈述性记忆区」作为归档
- 清空工作记忆区的「当前活跃任务看板」和「活跃子代理清单」

### 4. 收尾清理

- 删除临时文件、释放占用的资源
- 确认所有产出已正确存储到对应位置

#### 5. 最终交付准备

- 最终结果交付（向用户展示）

```markdown
✅ **任务完成**
- 结果：达成 / 未达成（原因：…）
- 产出：…
- 差异：计划 vs 实际的主要区别
- 同化：已更新 MEMORY.md 中的 [条目]
- 建议：下次类似的优化点（如有）
```

## 使用指南

### 调用方式

在得到用户反馈任务完成后，由主代理自动遵循，无需手动调用。
