## 工作流

### 工作流1：自行执行
```
### 1. 准备上下文
- **脚本**：名称、路径、使用方法
- **技能**：列表、调用方式、参数说明
- **密钥**：API密钥或访问凭证
- **环境**：技能、脚本所需要的工作环境
- **文件**：所有文件及其路径：包括最终文件、中间文件、中间脚本
- **执行步骤**：详细的执行步骤和流程
- 更新工作记忆任务看板状态为 active

### 2. 执行

### 3. 结果核验与处理

核验成功标准：
- 对照计划阶段定义的成功标准逐项验证
- 核查版本控制执行情况：备份、版本号更新、元数据更新
- 核查文件存储情况：路径、命名规范、权限设置
- 核查云文档同步情况：映射关系、权限设置
- 核查技能执行真实性：是否存在技能使用失败但编造成功结果的情况

核查结果处理：
- 核查通过 → 进入下一步
- 核查未通过 → 重新执行

### 4. 阶段结果汇总与反馈
- 更新 MEMORY.md 看板中对应任务的状态为 paused
- 汇总所有子任务执行结果
- 整理执行过程中的问题和解决方案
- 形成阶段执行结果报告


### 5. 执行反馈
- 执行结果摘要：[简要说明成果]
- 下阶段请求：请用户确认是需要对结果进行修改，还是确认任务完成


### 工作流2：标准监控循环
```
### 1. 准备上下文
- **脚本**：名称、路径、使用方法
- **技能**：列表、调用方式、参数说明
- **密钥**：API密钥或访问凭证
- **环境**：技能、脚本所需要的工作环境
- **文件**：所有文件及其路径：包括最终文件、中间文件、中间脚本
- **执行步骤**：详细的执行步骤和流程

### 2. 传递上下文与进行子代理执行
- 创建子代理并传递上下文
- 更新工作记忆任务看板与子代理追踪

### 3. 进度追踪与更新
- 偏差检查：
    - 未按计划使用指定技能/工具
    - 自行更换方案或使用计划外的其他方案
    - 实际进度与计划偏差超过10%
    - 发现偏差立即暂停执行，详细说明偏差情况，向用户请求指示

- 阻塞检查：监控任务处理时长，识别阻塞情况
    - 发现阻塞立即停止硬撑，向用户求助

### 4. 子代理结果同步
- 捕获子代理完成通告，提取结果摘要

### 5. 结果核验与处理

核验成功标准：
- 对照计划阶段定义的成功标准逐项验证
- 核查版本控制执行情况：备份、版本号更新、元数据更新
- 核查文件存储情况：路径、命名规范、权限设置
- 核查云文档同步情况：映射关系、权限设置
- 核查技能执行真实性：是否存在技能使用失败但编造成功结果的情况

核查结果处理：
- 核查通过 → 进入下一步
- 核查未通过 → 要求子代理修正问题，重新执行

### 6. 阶段结果汇总与反馈
- 更新 MEMORY.md 看板中对应任务的状态为 paused
- 汇总所有子任务执行结果
- 整理执行过程中的问题和解决方案
- 形成阶段执行结果报告


### 7. 执行反馈
- 执行结果（向用户展示），等待用户反馈
    - 场景1：偏差/阻塞
```markdown
⚠️ **阻塞告警**
- 当前步骤：第 X 步
- 偏差/阻塞内容：[具体描述]
- 已尝试的解决方案：[如有]
- **请用户提供下一步指导**
```

    - 场景2：子代理执行未通过
```markdown
❌ **执行异常**
- 子代理ID：[子代理key]
- 错误内容：[具体核查未通过原因]
- 已采取的修正措施：[如"已要求子代理重新执行"]
- 当前状态：[如"等待子代理修正结果"]
```

    - 场景3：执行成功
```markdown
👁️ **进度**：已完成 ①，正在执行 ②，预计下一步 ③。
- 已完成子任务：[列出已完成的任务ID和描述]
- 执行结果摘要：[简要说明成果]
- 下阶段请求：请用户确认是需要对结果进行修改，还是确认任务完成
```

## 使用指南

### 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `task_plan` | object | ✅ | 计划阶段输出的任务计划 |
| `current_time` | string | ✅ | 当前时间，格式 `YYYY-MM-DD HH:MM` |
| `subagent_outputs` | list | ❌ | 子代理已提交的产出 |

### 输出结果

| 输出项 | 格式 | 说明 |
|--------|------|------|
| `status_report` | Markdown | 当前任务状态报告 |
| `deviation_flag` | boolean | 是否检测到偏差 |
| `deviation_details` | object | 偏差类型、程度、影响范围 |

### 监控指标

| 指标 | 计算方法 | 阈值 | 触发动作 |
|------|----------|------|----------|
| 进度偏差率 | (计划进度-实际进度)/计划进度 | > 10% | 触发调节 |
| 时间消耗率 | 已用时间/预计总时间 | > 80% | 预警提醒 |
| 子代理活跃度 | 最后活跃时间距现在 | > 30min | 检查状态 |

### 状态更新格式

监控过程中，更新 `working_memory/subagent_tracker/SKILL.md`：

```markdown
### 读取任务状态
- 查询「活跃子代理清单」获取当前状态

### 更新任务状态
| 子代理key | 类型/角色 | 分配任务 | 状态 | 创建时间 | 最后活跃 | 备注 |
|-----------|-----------|----------|------|----------|----------|------|
| [key] | [类型] | [任务] | [更新] | [时间] | [现在] | [进度%] |
```
