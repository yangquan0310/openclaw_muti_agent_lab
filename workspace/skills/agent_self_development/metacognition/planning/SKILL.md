## 工作流

### 工作流1：制定完整任务计划

**步骤**：
1. **目标澄清与约束识别**
   - 明确任务的最终交付物
   - 识别时间、资源、能力等约束条件
   - 识别任务间的依赖关系

2. **任务层级拆解**
   - 将复杂任务拆分为可管理的子任务
   - 控制粒度：每个子任务可在合理时间内完成
   - 定义边界：明确子任务的输入、输出和验收标准

3. **子代理与工具分配**
   - 根据任务特性分配合适的子代理
   - 选择完成任务所需的工具/技能
   - 估算各子任务的执行时间

4. **创建任务看板记录**
   - 调用 `working_memory/subagent_tracker/SKILL.md` 创建记录
   - 状态设为 `active`

5. **制定执行计划**
   - 确定子任务的执行优先级和顺序
   - 定义关键里程碑和验收节点

## 使用指南

### 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `task_name` | string | ✅ | 任务名称 |
| `deliverables` | list | ✅ | 最终交付物清单 |
| `deadline` | string | ❌ | 截止时间，格式 `YYYY-MM-DD HH:MM` |
| `available_subagents` | list | ✅ | 可用子代理及其角色描述 |
| `dependencies` | list | ❌ | 任务间依赖关系 |

### 输出结果

| 输出项 | 格式 | 说明 |
|--------|------|------|
| `task_plan` | Markdown | 结构化任务计划文档 |
| `memory_record` | Markdown 表格 | 写入工作记忆的任务看板记录 |

### 任务计划模板

```markdown
## 任务计划: [任务名称]

### 基本信息
- 任务ID: [T001]
- 创建时间: [YYYY-MM-DD HH:MM]
- 预计完成: [YYYY-MM-DD HH:MM]

### 目标与交付物
- 最终目标: [描述]
- 交付物: [列表]

### 子任务清单
| 序号 | 子任务 | 负责人 | 预计时间 | 依赖 | 状态 |
|------|--------|--------|----------|------|------|
| 1 | [描述] | [代理] | [时长] | [依赖] | pending |

### 风险与应对
- 风险1: [描述] → 应对: [策略]

### 检查点
- [ ] 检查点1: [描述]
```

### 记忆记录格式

计划完成后，必须调用 `working_memory/subagent_tracker/SKILL.md` 创建任务记录：

```markdown
| 项目 | 任务 | 负责子代理key | 状态 | 创建时间 | 最后更新 | 备注 |
|------|------|---------------|------|----------|----------|------|
| [项目] | [任务] | [待分配] | active | [时间] | [时间] | [计划ID] |
```
