# metacognition

> 元认知模块 - 对认知的认知
> 基于「计划 → 监控 → 调节」三阶段闭环模型

---

## 概述

元认知（Metacognition）是指对自己的认知过程的认知和监控。本模块提供完整的元认知管理工具，帮助 Agent 实现：

- **计划（Planning）**：任务拆解、资源分配、策略选择
- **监控（Monitoring）**：进度跟踪、偏差检测、状态评估
- **调节（Regulation）**：策略调整、资源重配、计划修正

---

## 子模块

| 子模块 | 路径 | 功能描述 |
|--------|------|----------|
| 计划阶段 | `planning/SKILL.md` | 任务拆解、计划制定 |
| 监控阶段 | `monitoring/SKILL.md` | 进度监控、状态跟踪 |
| 调节阶段 | `regulation/SKILL.md` | 策略调节、计划修正 |

---

## 三阶段闭环模型

```
┌─────────┐     ┌─────────┐     ┌─────────┐
│  计划   │ ──→ │  监控   │ ──→ │  调节   │
│Planning │     │Monitoring│    │Regulation│
└─────────┘     └─────────┘     └────┬────┘
      ↑                              │
      └──────────────────────────────┘
              (反馈循环)
```

## 使用场景

### 场景1：复杂任务执行前
**使用**：`planning/SKILL.md`  
**操作**：拆解任务、制定执行计划、分配子代理

### 场景2：任务执行中
**使用**：`monitoring/SKILL.md`  
**操作**：跟踪进度、检测偏差、记录状态

### 场景3：发现执行偏差
**使用**：`regulation/SKILL.md`  
**操作**：调整策略、修正计划、重新分配资源

---

## 与 working_memory 的关系

```
元认知 (Metacognition)
    ├── 计划 → 写入 working_memory（创建任务记录）
    ├── 监控 → 读取 working_memory（获取任务状态）
    └── 调节 → 更新 working_memory（修改任务状态）
```
