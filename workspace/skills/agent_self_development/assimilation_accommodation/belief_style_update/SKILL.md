# belief_style_update

> 同化与顺应 - 信念与风格更新
> 更新工作信念和工作风格

---

## 概述

信念（Beliefs）是 Agent 对工作的基本看法和价值观，风格（Style）是 Agent 的工作方式和偏好。当 Agent 通过实践发现原有信念或风格需要调整时，需要更新这些定义。

---

## 更新范围

### 1. 工作信念更新

**信念类型**：
- **工作信念**：对工作的基本看法
- **价值观优先级**：各项价值的优先排序
- **安全红线**：不可妥协的原则

**更新场景**：
- 实践中发现原有信念不适用
- 用户明确反馈价值观冲突
- 团队规范调整

### 2. 工作风格更新

**风格维度**：
- **交互风格**：与用户和团队的交互方式
- **文档风格**：文档写作的习惯和规范
- **代码风格**：代码编写的方式和偏好
- **任务执行风格**：执行任务的方法和习惯

**更新场景**：
- 用户明确反馈风格偏好
- 实践中发现风格效率问题
- 团队协作需要统一风格

---

## 更新流程

```
发展日记检测到信念/风格变化
    ↓
[步骤1] 分析变化原因
    ├── 用户反馈 → 确认用户偏好
    ├── 实践发现 → 验证新方式有效性
    └── 团队需要 → 确认团队规范
    ↓
[步骤2] 评估影响范围
    ├── 仅影响当前 Agent
    ├── 影响多个 Agent（需同步）
    └── 影响整个实验室（需规范更新）
    ↓
[步骤3] 更新 SOUL.md
    ├── 修改「信念」章节
    └── 修改「风格」章节
    ↓
[步骤4] 同步到 lab_repository
    └── 更新 lab_repository/SOUL.md
    ↓
[步骤5] 通知相关方（如影响其他 Agent）
    ↓
[步骤6] 记录更新日志
```
