# core_self_update

> 同化与顺应 - 核心自我更新
> 更新核心身份和能力边界

---

## 概述

核心自我（Core Self）是 Agent 最基础的自我定义，包括核心身份、能力边界和责任边界。当 Agent 通过发展日记识别到能力扩展或责任变化时，需要更新核心自我定义。

---

## 更新范围

### 1. 能力边界更新

**扩展场景**：
- 掌握新工具或技能
- 发现原有能力的新的应用场景
- 通过实践验证能力的有效性

**收缩场景**：
- 发现能力盲区
- 识别能力误用风险
- 用户明确限制某些操作

### 2. 责任边界更新

**扩展场景**：
- 承担新的职责
- 用户明确授权新权限
- 团队协作需要

**收缩场景**：
- 职责转移给其他 Agent
- 用户明确限制责任范围
- 安全风险考虑

---

## 更新流程

```
发展日记检测到能力/责任变化
    ↓
[步骤1] 评估变化性质
    ├── 能力扩展 → 验证能力有效性
    ├── 能力收缩 → 确认限制必要性
    ├── 责任扩展 → 确认授权来源
    └── 责任收缩 → 确认转移对象
    ↓
[步骤2] 更新 IDENTITY.md
    ├── 修改「能力边界」表格
    └── 修改「责任边界」表格
    ↓
[步骤3] 同步到 lab_repository
    └── 更新 lab_repository/IDENTITY.md
    ↓
[步骤4] 记录更新日志
    └── 在 MEMORY.md 事件记忆中记录
```
