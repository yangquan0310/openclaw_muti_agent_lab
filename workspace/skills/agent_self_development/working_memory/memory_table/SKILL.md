# memory_table

> 工作记忆 - 记忆表管理
> 维护工作记忆的表格结构和数据完整性

---

## 概述

记忆表是工作记忆的核心数据结构，用于存储和管理当前活跃的任务和子代理信息。本技能负责维护记忆表的格式规范、数据完整性和定期清理。

---

## 核心功能

### 1. 表格结构维护

- **当前活跃任务看板**：维护任务追踪表格
- **活跃子代理清单**：维护子代理追踪表格
- **表头格式检查**：确保表格列名和顺序正确

### 2. 数据完整性检查

- **必填字段检查**：确保关键字段不为空
- **时间格式检查**：确保时间戳格式统一
- **状态值检查**：确保状态值在允许范围内

### 3. 定时清理

- **归档 completed 任务**：将已完成任务转移到「事件记忆」
- **删除 killed 任务**：直接删除异常终止的任务
- **清理空行**：删除表格中的空行

---

## 表格结构规范

### 当前活跃任务看板

```markdown
| 项目 | 任务 | 负责子代理key | 状态 | 创建时间 | 最后更新 | 备注 |
|------|------|---------------|------|----------|----------|------|
```

**字段说明**：
| 字段 | 必填 | 格式 | 示例 |
|------|------|------|------|
| 项目 | 是 | 字符串 | 数字化存储与自传体记忆 |
| 任务 | 是 | 字符串 | 文献检索与综述 |
| 负责子代理key | 是 | agent:xxx:subagent:uuid | agent:steward:subagent:xxx |
| 状态 | 是 | active/paused/completed/killed | active |
| 创建时间 | 是 | YYYY-MM-DD HH:MM | 2026-04-17 10:30 |
| 最后更新 | 是 | YYYY-MM-DD HH:MM | 2026-04-17 14:00 |
| 备注 | 否 | 字符串 | 进度50% |

### 活跃子代理清单

```markdown
| 子代理key | 类型/角色 | 分配任务 | 状态 | 创建时间 | 最后活跃 | 备注 |
|-----------|-----------|----------|------|----------|----------|------|
```

**字段说明**：
| 字段 | 必填 | 格式 | 示例 |
|------|------|------|------|
| 子代理key | 是 | agent:xxx:subagent:uuid | agent:steward:subagent:xxx |
| 类型/角色 | 是 | 字符串 | 文献检索助手 |
| 分配任务 | 是 | 字符串 | 检索自传体记忆相关文献 |
| 状态 | 是 | active/paused/completed/killed | active |
| 创建时间 | 是 | YYYY-MM-DD HH:MM | 2026-04-17 10:30 |
| 最后活跃 | 是 | YYYY-MM-DD HH:MM | 2026-04-17 14:00 |
| 备注 | 否 | 字符串 | 已检索50篇 |

## 定时清理流程

```
每日 04:00 触发
    ↓
[步骤1] 读取「活跃子代理清单」
    ↓
[步骤2] 筛选状态为 completed 的行
    ↓
[步骤3] 归档到「事件记忆」表格
    ├── 日期: 当前日期
    ├── 事件: 子代理任务归档
    ├── 涉及实体: 子代理key
    ├── 结果: 任务完成
    └── 日志位置: ~/实验室仓库/日志文件/YYYY-MM-DD/
    ↓
[步骤4] 筛选状态为 killed 的行
    ↓
[步骤5] 直接删除 killed 行（不归档）
    ↓
[步骤6] 从清单中删除 completed 和 killed 行
    ↓
[步骤7] 记录清理日志
```

---

## 与 assimilation_accommodation 的关系

定时清理后的归档数据会进入「事件记忆」，成为长期陈述性记忆的一部分：

```
working_memory (短期)
    └── completed 任务
            ↓ 归档
assimilation_accommodation (长期)
    └── 事件记忆表
```
