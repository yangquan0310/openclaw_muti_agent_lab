# working_memory

> 工作记忆模块 - 管理当前活跃任务和子代理状态
> 基于 Baddeley 工作记忆模型

---

## 概述

工作记忆（Working Memory）是 Agent 的短期记忆系统，用于存储和管理当前活跃的任务、子代理及其状态。基于 Baddeley 的工作记忆模型设计，确保 Agent 能够有效地追踪和协调多个并发任务。

---

## 子模块

| 子模块 | 路径 | 功能描述 |
|--------|------|----------|
| 记忆表管理 | `memory_table/SKILL.md` | 管理工作记忆表格结构 |
| 子代理追踪 | `subagent_tracker/SKILL.md` | 追踪子代理状态和任务分配 |

## 工作记忆模型

```
┌─────────────────────────────────────────┐
│           中央执行系统                  │
│        (Central Executive)              │
│         - 任务协调与分配                 │
├─────────────┬─────────────┬─────────────┤
│   任务存储   │  子代理存储  │   状态缓存   │
│  (记忆表)   │  (追踪器)   │  (临时信息)  │
└─────────────┴─────────────┴─────────────┘
```

---

## 核心功能

### 1. 任务存储

- **当前活跃任务看板**：记录所有正在执行的任务
- **任务状态管理**：pending / active / paused / completed / killed
- **任务属性记录**：项目、任务描述、负责人、时间戳等

### 2. 子代理追踪

- **活跃子代理清单**：记录所有活跃的子代理
- **子代理状态监控**：跟踪子代理的执行状态
- **任务分配映射**：记录子代理与任务的对应关系

### 3. 状态同步

- **与 metacognition 同步**：计划/监控/调节阶段的状态更新
- **与 assimilation_accommodation 同步**：任务完成后的归档
- **定时清理**：每日归档 completed/killed 任务

## 使用场景

### 场景1：任务创建
**使用**：`subagent_tracker/SKILL.md`  
**操作**：在「当前活跃任务看板」和「活跃子代理清单」中创建新记录

### 场景2：状态更新
**使用**：`subagent_tracker/SKILL.md`  
**操作**：更新任务状态、子代理最后活跃时间

### 场景3：任务归档
**使用**：`memory_table/SKILL.md` + `subagent_tracker/SKILL.md`  
**操作**：将 completed 任务归档到「事件记忆」，从清单中删除

---

## 状态说明

| 状态 | 含义 | 处理方式 |
|------|------|----------|
| `active` | 正在执行 | 保留在清单中 |
| `paused` | 暂停等待 | 保留在清单中 |
| `completed` | 已完成 | 归档到事件记忆后删除 |
| `killed` | 被终止 | 直接删除，不归档 |

---

## 定时清理任务

- **执行时间**：每日 04:00 (Asia/Shanghai)
- **执行方式**：子代理执行
- **任务内容**：
  1. 扫描「活跃子代理清单」
  2. 将 `completed` 状态任务归档到「事件记忆」
  3. 直接删除 `killed` 状态任务
  4. 从清单中删除已处理的任务
  5. 记录清理日志
