#### S1：论文审稿

##### 触发条件
需要审查论文时，按照审稿论文检查清单.md逐条核查论文中的问题

##### 依赖
- 审稿论文检查清单.md（位于项目模板文件夹）
- 管理元数据.md（更新元数据.json格式规范）
- docx技能（读取Word文档）

##### 输入
1. **论文文件路径**：待审查的论文文件路径（.docx或.md格式）
2. **项目路径**：项目根目录路径
3. **轮次**：审稿轮次（如R1、R2）
4. **检查清单路径**：审稿论文检查清单.md的路径

##### 步骤

1. **读取论文文件**
   - 使用docx技能读取Word文档内容
   - 或读取已转换的.md文件
   - 提取论文标题、作者、摘要、正文等关键部分

2. **创建审稿意见文件**
   - 文件名格式：`{论文标题}_{轮次}.md`
   - 位置：`{项目路径}/审稿意见/`
   - 文件结构基于审稿论文检查清单.md模板

3. **逐条核查检查清单**
   - 按照审稿论文检查清单.md的模块顺序审查
   - 每个检查项填写状态（合格/不合格/完整/缺失）
   - 发现问题时填写具体问题描述
   - 核查模块包括：
     - 基本信息
     - 投稿前自检（伦理、版权）
     - 题目与作者信息
     - 摘要检查（中英文）
     - 正文结构（引言、方法、结果、讨论）
     - 参考文献
     - 图表规范
     - 格式规范
     - 学术规范

4. **撰写审稿意见**
   - 按优先级排序问题（高/中/低）
   - 提供具体修改建议
   - 给出综合评价和审稿结论

5. **更新元数据.json**
   - 在元数据.json中添加审稿记录
   - JSON格式：
   ```json
   {
     "review_records": [
       {
         "paper_title": "论文标题",
         "round": "R1",
         "review_date": "2026-04-11",
         "reviewer": "reviewer",
         "status": "审稿中/已完成",
         "opinion_file": "审稿意见/{论文标题}_R1.md",
         "score": {
           "选题价值": 4,
           "研究设计": 4,
           "方法学": 3,
           "数据分析": 4,
           "结果解释": 4,
           "写作规范": 3,
           "学术规范": 5
         },
         "overall_rating": "良好",
         "conclusion": "小修后录用",
         "major_issues": 2,
         "minor_issues": 5
       }
     ]
   }
   ```

6. **同步到管理元数据.md**
   - 将JSON格式同步到管理元数据.md文档中
   - 保持格式一致性

##### 输出
1. **审稿意见文件**：`{项目路径}/审稿意见/{论文标题}_{轮次}.md`
   - 完整的检查清单填写结果
   - 问题列表（按优先级排序）
   - 详细修改建议
   - 综合评价和审稿结论

2. **元数据更新**：`{项目路径}/元数据.json`
   - 添加review_records数组
   - 记录审稿历史

3. **管理元数据.md同步**：更新JSON格式示例
