### 1.3 工作记忆使用规则

工作记忆使用规则

##### 初始化规则（计划阶段）
- 每次复杂任务开始前，主代理必须先加载此看板，检查是否有未完成任务
- 拆解子任务后，立即添加新行到「当前活跃任务看板」，状态为`active`
- 为每个任务分配唯一任务ID（格式：T001, T002, ...）

##### 执行规则（监控阶段）
- 创建子代理后，将子代理key和分配的任务填入「活跃子代理清单」
- 每完成一个子任务 → 更新任务状态为`completed`，结果摘要填入备注
- 当需要并行执行独立子任务时 → 使用子代理，状态改为`active`
- 子代理完成（收到通告）后，将任务状态改为`completed`，结果摘要填入备注
- 发现偏离计划超过 10% → 暂停并更新状态为`paused`，请求用户指示

##### 清理规则（调节阶段）
- 任务全部结束后，将任务状态改为`completed`
- **只将`completed`状态的任务归档到「事件记忆」表格**（包含：日期、事件、涉及实体、结果、日志位置）
- **`killed`状态的任务直接删除，不归档**（异常终止的任务不进入历史记录）
- **从「活跃子代理清单」中删除`completed`和`killed`状态的任务**
- 只保留`active`和`paused`状态的任务在清单中
- 保留`active`状态的任务用于持续监控

##### 定时清理任务
- **执行时间**：每日凌晨04:00（Asia/Shanghai）
- **执行方式**：子代理执行
- **任务内容**：
  1. 扫描「活跃子代理清单」
  2. **只将`completed`状态的任务归档到「事件记忆」**（killed状态直接删除）
  3. 从清单中删除`completed`和`killed`状态的任务
  4. 记录清理日志（区分归档和删除的操作）

##### 状态说明与处理规则

| 状态 | 处理方式 | 说明 |
|------|----------|------|
| `active` | **保留在清单中** | 子代理正在工作或待命（执行中） |
| `paused` | **保留在清单中** | 任务暂停，等待用户反馈 |
| `completed` | **归档到事件记忆后删除** | 任务正常完成，归档到历史记录 |
| `killed` | **直接删除，不归档** | 任务异常终止，不进入历史记录 |

##### 一致性维护
- 确保「当前活跃任务看板」和「活跃子代理清单」状态一致
- **每日凌晨04:00自动清理**：`completed`任务归档到事件记忆，`killed`任务直接删除
- 每次任务结束后执行闭环自检清单

## 五、程序性记忆（Procedural Memory）

### 脚本（Scripts）

#### P1: 执行物理建模任务
```
触发条件: 收到物理建模或理论分析任务
执行方式: 创建子代理执行
依赖: 通用脚本技能（文档管理、版本控制等）
输入: 物理问题描述、建模目标、约束条件
输出: 物理模型、理论分析报告、物理解释

步骤:
  1. 问题理解
     - 明确物理现象
     - 识别关键变量
     - 确定建模目标

  2. 物理分析
     - 识别主导物理机制
     - 确定适用的物理定律
     - 列出相关守恒量

  3. 模型建立
     - 选择坐标系和参考系
     - 建立几何模型
     - 定义物理量和参数
     - 写出基本方程

  4. 简化与近似
     - 识别小参数
     - 确定主导项
     - 忽略次要效应

  5. 模型验证
     - 量纲检查
     - 极限情况检验
     - 与已知结果对比

  6. 数学求解
     - 选择求解方法
     - 进行数学推导
     - 处理边界条件

  7. 结果分析
     - 分析解的物理意义
     - 讨论参数依赖性
     - 识别特征尺度

  8. 物理解释
     - 建立物理图像
     - 解释物理机制
     - 预测可观测效应

  9. 生成分析报告
     - 整理建模过程
     - 记录关键步骤
     - 撰写分析结果

  10. 记录到工作日志
```

#### P2: 理论分析与推导
```
触发条件: 需要物理理论分析
执行方式: 直接执行或创建子代理
依赖: 物理学理论基础
输入: 理论问题、已知条件、推导目标
输出: 理论推导过程、物理解释、最终公式

步骤:
  1. 理论框架识别
     - 确定问题所属理论体系
     - 选择适当的理论框架
     - 列出基本假设

  2. 数学推导
     - 从基本原理出发
     - 应用数学工具
     - 逐步推导

  3. 物理意义阐释
     - 解释每个步骤的物理含义
     - 连接数学与物理概念
     - 强调关键物理思想

  4. 极限情况分析
     - 分析特殊极限
     - 验证物理合理性
     - 与经典结果对比

  5. 误差与近似评估
     - 评估推导中的近似
     - 估计误差范围
     - 讨论适用条件

  6. 结果呈现
     - 整理推导过程
     - 突出关键步骤
     - 提供物理解释
```

### 条件-行动规则（If-Then Rules）

| 条件 | 行动 |
|------|------|
| 需要物理建模 | 调用 P1 执行物理建模任务 |
| 需要理论推导 | 调用 P2 理论分析与推导 |
| 需要分析物理问题类型 | 调用 P3 物理问题分类与识别 |
| 涉及力学问题 | 优先使用牛顿力学或分析力学框架 |

| 问题复杂度高 | 优先考虑简化模型，逐步增加复杂度 |

| 需要数学协助 | 建议与数学家协作 |
