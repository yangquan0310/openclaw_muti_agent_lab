### 工作记忆使用规则

#### 初始化规则（计划阶段）
- 每次复杂任务开始前，主代理必须先加载此看板，检查是否有未完成任务
- 拆解子任务后，立即添加新行到「当前活跃任务看板」，状态为`active`
- 为每个任务分配唯一任务ID（格式：T001, T002, ...）

#### 执行规则（监控阶段）
- 创建子代理后，将子代理key和分配的任务填入「活跃子代理清单」
- 每完成一个子任务 → 更新任务状态为`completed`，结果摘要填入备注
- 当需要并行执行独立子任务时 → 使用子代理，状态改为`active`
- 子代理完成（收到通告）后，将任务状态改为`completed`，结果摘要填入备注
- 发现偏离计划超过 10% → 暂停并更新状态为`paused`，请求用户指示

#### 清理规则（调节阶段）
- 任务全部结束后，将任务状态改为`completed`
- 将本次任务看板中的历史记录移动到陈述性记忆区作为归档
- 清空工作记忆区的已完成任务
- 保留`active`状态的任务用于持续监控

#### 状态说明
- `active`: 子代理正在工作或待命（执行中）
- `paused`: 任务暂停，等待用户反馈
- `completed`: 任务已完成，子代理已归档

#### 一致性维护
- 确保「当前活跃任务看板」和「活跃子代理清单」状态一致
- 每日清理已归档超过7天的历史记录
- 每次任务结束后执行闭环自检清单

## 🛠️ 程序性记忆区

### 脚本1：科研项目管理

**触发条件**：
- 用户明确说"创建科研项目"、"新建科研项目"
- 用户上传研究方案或项目申请书
- 用户询问"如何管理科研项目"

**执行方式**：
- 主代理直接执行（简单任务）或创建子代理（复杂项目）

**依赖**：
- `~/教研室仓库/`目录存在
- 项目模板文件可用

**输入**：
- 项目名称
- 研究领域
- 研究目标（可选）
- 团队成员（可选）
- 时间规划（可选）

**输出**：
- 项目文件夹结构
- 项目元数据文件（project_metadata.json）
- 项目README.md说明文件
- 项目进度跟踪表

**步骤**：
1. **收集项目信息**：询问用户获取项目基本信息
2. **创建项目文件夹**：在`~/教研室仓库/项目文件/`下创建`YYYY-MM-DD_项目名/`文件夹
3. **建立标准结构**：
   ```
   YYYY-MM-DD_项目名/
   ├── 原始文档/           # 用户上传的原始文档
   ├── 原始数据/           # 原始研究数据
   ├── 文献资料/           # 相关文献PDF和笔记
   ├── 分析代码/           # 数据分析代码
   ├── 论文草稿/           # 论文草稿
   ├── 会议材料/           # 会议摘要、PPT等
   ├── 经费管理/           # 经费预算和报销记录
   ├── 元数据.json         # 项目元数据
   └── README.md           # 项目说明
   ```
4. **初始化元数据**：创建`元数据.json`，包含项目信息
5. **创建进度跟踪表**：初始化项目进度跟踪表
6. **生成项目报告**：创建初始项目报告文档
7. **确认完成**：向用户展示创建的项目结构

### 脚本2：学术文献整理

**触发条件**：
- 用户说"整理文献"、"文献检索"、"查找参考文献"
- 用户上传文献PDF文件
- 用户询问"如何管理参考文献"

**执行方式**：
- 主代理直接执行（简单整理）或创建子代理（大量文献）

**依赖**：
- 网络连接（用于文献检索）
- 文献管理工具知识

**输入**：
- 研究主题/关键词
- 文献PDF文件（可选）
- 文献管理格式要求（可选）

**输出**：
- 文献列表（带摘要和引用信息）
- 文献分类文件夹
- 文献阅读笔记
- 参考文献格式文件

**步骤**：
1. **明确需求**：询问用户具体的研究领域和关键词
2. **文献检索**：
   - 使用学术搜索引擎检索相关文献
   - 筛选高质量期刊和会议论文
   - 记录检索结果（标题、作者、年份、摘要）
3. **文献获取**：
   - 提供文献下载链接或指导
   - 协助用户获取全文PDF
4. **文献整理**：
   - 按主题/时间/重要性分类
   - 创建文献信息表格
   - 提取关键信息（研究问题、方法、结论）
5. **笔记制作**：
   - 为每篇重要文献创建阅读笔记
   - 记录核心观点、研究方法、创新点
   - 标注文献质量等级
6. **引用管理**：
   - 生成标准格式的参考文献列表
   - 提供不同引用格式（APA、MLA、Chicago等）
   - 创建参考文献管理文件
7. **交付成果**：向用户提供完整的文献整理包

### 脚本3：科研进度跟踪

**触发条件**：
- 用户说"项目进度"、"进度报告"、"跟踪进展"
- 定期报告需求（周报、月报）
- 项目中期/结题检查

**执行方式**：
- 主代理直接执行

**依赖**：
- 项目文件夹存在
- 项目元数据完整

**输入**：
- 项目名称/路径
- 跟踪时间段
- 进度指标（可选）

**输出**：
- 进度报告文档
- 进度可视化图表
- 风险预警信息
- 下一步建议

**步骤**：
1. **项目识别**：确定要跟踪的项目
2. **数据收集**：
   - 读取项目元数据和历史记录
   - 检查各文件夹内容更新情况
   - 收集团队成员进展反馈
3. **进度分析**：
   - 对比计划与实际进展
   - 计算完成百分比
   - 识别延迟和风险
4. **报告生成**：
   - 创建结构化进度报告
   - 包含：已完成工作、进行中工作、未开始工作
   - 添加时间线图和甘特图
5. **风险预警**：
   - 识别可能影响进度的风险
   - 提供风险缓解建议
   - 标注需要关注的问题
6. **建议制定**：
   - 基于当前进度提出调整建议
   - 制定下一步行动计划
   - 分配资源和时间
7. **报告交付**：向用户提供完整的进度跟踪报告
