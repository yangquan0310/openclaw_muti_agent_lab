### 1.3 工作记忆使用规则

#### 初始化规则（计划阶段）
- 每次复杂任务开始前，主代理必须先加载此看板，检查是否有未完成任务
- 拆解子任务后，立即添加新行到「当前活跃任务看板」，状态为`active`
- 为每个任务分配唯一任务ID（格式：T001, T002, ...）

#### 执行规则（监控阶段）
- 创建子代理后，将子代理key和分配的任务填入「活跃子代理清单」
- 每完成一个子任务 → 更新任务状态为`completed`，结果摘要填入备注
- 当需要并行执行独立子任务时 → 使用子代理，状态改为`active`
- 子代理完成（收到通告）后，将任务状态改为`completed`，结果摘要填入备注
- 发现偏离计划超过 10% → 暂停并更新状态为`paused`，请求用户指示

#### 清理规则（调节阶段）
- 任务全部结束后，将任务状态改为`completed`
- **只将`completed`状态的任务归档到「事件记忆」表格**（包含：日期、事件、涉及实体、结果、日志位置）
- **`killed`状态的任务直接删除，不归档**（异常终止的任务不进入历史记录）
- **从「活跃子代理清单」中删除`completed`和`killed`状态的任务**
- 只保留`active`和`paused`状态的任务在清单中
- 保留`active`状态的任务用于持续监控

#### 定时清理任务
- **执行时间**：每日凌晨04:00（Asia/Shanghai）
- **执行方式**：子代理执行
- **任务内容**：
  1. 扫描「活跃子代理清单」
  2. **只将`completed`状态的任务归档到「事件记忆」**（killed状态直接删除）
  3. 从清单中删除`completed`和`killed`状态的任务
  4. 记录清理日志（区分归档和删除的操作）

#### 状态说明与处理规则

| 状态 | 处理方式 | 说明 |
|------|----------|------|
| `active` | **保留在清单中** | 子代理正在工作或待命（执行中） |
| `paused` | **保留在清单中** | 任务暂停，等待用户反馈 |
| `completed` | **归档到事件记忆后删除** | 任务正常完成，归档到历史记录 |
| `killed` | **直接删除，不归档** | 任务异常终止，不进入历史记录 |

#### 一致性维护
- 确保「当前活跃任务看板」和「活跃子代理清单」状态一致
- **每日凌晨04:00自动清理**：`completed`任务归档到事件记忆，`killed`任务直接删除
- 每次任务结束后执行闭环自检清单

### 知识网络

#### 自传体记忆研究语义网络

**核心概念**：autobiographical memory（自传体记忆）

**同义词/近义词**：
- episodic memory（情景记忆）
- personal memory（个人记忆）
- life narrative（生命叙事）
- personal event memory（个人事件记忆）

**下位概念/细分领域**：
- self-defining memory（自我定义记忆）
- flashbulb memory（闪光灯记忆）
- life story（生命故事）
- earliest childhood memory（最早童年记忆）
- reminiscence bump（回忆隆起）

**相关概念/上位概念**：
- memory consolidation（记忆巩固）
- memory retrieval（记忆提取）
- episodic future thinking（情景未来思维）
- mental time travel（心理时间旅行）
- self-memory system（自我记忆系统）

**关键作者**：
- Conway（自我记忆系统理论）
- Pillemer（个人事件记忆）
- Nelson（生命故事发展）
- Fivush（叙事与自传体记忆）
- Tulving（情景记忆）

**核心期刊**：
- Memory
- Memory & Cognition
- Psychological Review
- Journal of Experimental Psychology
- Cognition & Emotion

**7个研究主题**：
1. 自传体记忆基础（定义、结构、模型）
2. 自传体记忆功能（自我功能、社会功能、指导功能）
3. 自传体记忆的自我参照编码（自我参照效应）
4. 自传体记忆的主动遗忘（定向遗忘、有意遗忘）
5. 自传体记忆的系统性巩固（记忆巩固、睡眠与记忆）
6. 自传体记忆的生成性提取（精细提取、生成性检索）
7. 数字化使用对自传体记忆的影响（拍照效应、Google效应、认知卸载）

- 检索陈述性记忆中的知识网络，明确研究主题的领域归属
- **扩展关键词**：提取核心关键词的同义词、下位词、相关概念
- **构建检索词组合**：使用多个不同的检索词组合覆盖主题全貌
- 梳理该领域的关键作者、经典研究、核心期刊
- 确定检索的纳入排除标准、时间范围、文献类型要求

### 步骤2：文献检索与收集

**禁止规则（检索阶段必须遵守）**：
- 严禁使用Semantic Scholar和Zotero之外的任何检索方式
- 严禁编造、杜撰文献信息或检索结果
- 检索出错必须如实报告错误类型和原因，不得隐瞒

**检索流程**：
- **Zotero本地检索**：
  - **按关键词检索**
- **Semantic Scholar网络检索**：
  - **使用多个不同的检索词组合进行多轮检索**
  - **奠基文献**：发表最早且高引用，最多10篇
  - **重要文献**：近20年引用50+，全部保留
  - **新近文献**：近5年实证研究（实验法/问卷法），最多30篇
- 合并检索结果并去重，按照影响因子、引用量、相关性排序
- 所有检索过程必须留痕，记录检索词、检索时间、返回数量

### 步骤3：文献分级与整理
- **奠基文献**：发表最早且高引用
- **重要文献**：引用量50+以上
- **一般文献**：引用量<50
- 补充DOI信息，标记文献分级

### 步骤4：文献统计与分析（仅描述事实，不归纳推断）
- 主题分布统计
- 年份分布统计
- **文献类型分布：综述文献 vs 实证研究**
- 核心作者统计（发文数量）
- 核心期刊统计（载文数量）

### 步骤5：检索报告撰写
- 报告仅描述事实与统计数据，不进行归纳推断
- 报告包含以下章节（仅事实描述与统计数据）：
1. **检索概况**：基本统计、详细检索记录、复现说明
2. **主题分布**：各主题文献数量及占比
3. **文献分布**：
   - 年份分布
   - **文献类型分布（综述 vs 实证）**
   - **文献分级分布（奠基/重要/一般）**
4. **核心作者**：高频作者列表及发文数量
5. **核心期刊**：高频期刊列表及载文数量
6. **参考文献**：
   - **奠基文献 APA格式**
   - **重要文献 APA格式**
   - **新近文献 APA格式**

### 步骤6：文档管理与交付
- 按照「创建文件.md」创建检索报告文档（终稿/目录）
- 按照「控制版本.md」进行版本控制
- 返回本地路径、版本信息、云文档链接（如已上传）

#### S2-执行多主题并行文献检索任务

**触发条件**: 需要检索多个主题的文献（≥3个主题）

**执行方式**: 主代理协调 + 多个子代理并行执行（最多2个子代理同时运行）

**依赖**: 
- semantic-scholar-mcp 技能
- 工作记忆追踪系统
- S1脚本（单主题检索标准）

**输入**: 
- 主题列表（每个主题包含名称和检索关键词）
- 每主题文献分级标准（奠基/重要/新近）
- 并行度限制（默认最多2个子代理）

**步骤**:
1. **工作记忆初始化**
   - 在MEMORY.md工作记忆区创建子代理任务追踪表
   - 为每个主题分配子代理ID占位符
   - 设置状态为"⏳ 待执行"

2. **分批创建子代理**
   - 按并行度限制（2个）分批创建子代理
   - **向子代理传递完整上下文**：
     - API Key: `SEMANTIC_SCHOLAR_API_KEY`
     - 技能: `semantic-scholar-mcp` (使用 `mcporter call semantic-scholar.semantic_scholar_search`)
     - 检索标准: 奠基(最早+高引用,≤3篇)/重要(近20年引用500+)/新近(近3年实证,≤30篇)
     - 主题名称和检索关键词
     - 输出文件路径: `/tmp/topic{N}_{name}.json`
   - 每个子代理执行S1脚本检索一个主题
   - 更新工作记忆状态为"🔄 执行中"

3. **监控与调度**
   - 监听子代理完成事件
   - 每完成一个主题，立即创建下一个主题的子代理
   - 更新工作记忆状态和结果摘要

4. **结果整合**
   - 所有主题完成后，读取各主题索引文件
   - 去重、分级、整合到统一知识库
   - 生成综合检索报告

5. **工作记忆清理**
   - 标记所有子代理任务为"✅ 已完成"
   - 归档到事件记忆
   - 更新程序性记忆（如有新模式）

**输出**:
- 各主题独立索引文件（topic{N}_{name}.json）
- 整合后的知识库（index.json）
- 综合检索报告
- 工作记忆更新记录
