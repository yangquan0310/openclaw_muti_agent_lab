### 1.3 工作记忆使用规则

#### 初始化规则（计划阶段）
- 每次复杂任务开始前，主代理必须先加载此看板，检查是否有未完成任务
- 拆解子任务后，立即添加新行到「当前活跃任务看板」，状态为`active`
- 为每个任务分配唯一任务ID（格式：T001, T002, ...）

#### 执行规则（监控阶段）
- 创建子代理后，将子代理key和分配的任务填入「活跃子代理清单」
- 每完成一个子任务 → 更新任务状态为`completed`，结果摘要填入备注
- 当需要并行执行独立子任务时 → 使用子代理，状态改为`active`
- 子代理完成（收到通告）后，将任务状态改为`completed`，结果摘要填入备注
- 发现偏离计划超过 10% → 暂停并更新状态为`paused`，请求用户指示


#### 清理规则（调节阶段）
- 任务全部结束后，将任务状态改为`completed`
- **只将`completed`状态的任务归档到「事件记忆」表格**（包含：日期、事件、涉及实体、结果、日志位置）
- **`killed`状态的任务直接删除，不归档**（异常终止的任务不进入历史记录）
- **从「活跃子代理清单」中删除`completed`和`killed`状态的任务**
- 只保留`active`和`paused`状态的任务在清单中
- 保留`active`状态的任务用于持续监控

#### 定时清理任务
- **执行时间**：每日凌晨04:00（Asia/Shanghai）
- **执行方式**：子代理执行
- **任务内容**：
  1. 扫描「活跃子代理清单」
  2. **只将`completed`状态的任务归档到「事件记忆」**（killed状态直接删除）
  3. 从清单中删除`completed`和`killed`状态的任务
  4. 记录清理日志（区分归档和删除的操作）


#### 状态说明
- `active`: 子代理正在工作或待命（执行中）
- `paused`: 任务暂停，等待用户反馈
- `completed`: 任务已完成，子代理已归档

#### 一致性维护
- 确保「当前活跃任务看板」和「活跃子代理清单」状态一致
- 每日清理已归档超过7天的历史记录
- 每次任务结束后执行闭环自检清单

### 知识网络

#### 自传体记忆研究语义网络

**核心概念**：autobiographical memory（自传体记忆）

**同义词/近义词**：
- episodic memory（情景记忆）
- personal memory（个人记忆）
- life narrative（生命叙事）
- personal event memory（个人事件记忆）

**下位概念/细分领域**：
- self-defining memory（自我定义记忆）
- flashbulb memory（闪光灯记忆）
- life story（生命故事）
- earliest childhood memory（最早童年记忆）
- reminiscence bump（回忆隆起）

**相关概念/上位概念**：
- memory consolidation（记忆巩固）
- memory retrieval（记忆提取）
- episodic future thinking（情景未来思维）
- mental time travel（心理时间旅行）
- self-memory system（自我记忆系统）

**关键作者**：
- Conway（自我记忆系统理论）
- Pillemer（个人事件记忆）
- Nelson（生命故事发展）
- Fivush（叙事与自传体记忆）
- Tulving（情景记忆）

**核心期刊**：
- Memory
- Memory & Cognition
- Psychological Review
- Journal of Experimental Psychology
- Cognition & Emotion

**7个研究主题**：
1. 自传体记忆基础（定义、结构、模型）
2. 自传体记忆功能（自我功能、社会功能、指导功能）
3. 自传体记忆的自我参照编码（自我参照效应）
4. 自传体记忆的主动遗忘（定向遗忘、有意遗忘）
5. 自传体记忆的系统性巩固（记忆巩固、睡眠与记忆）
6. 自传体记忆的生成性提取（精细提取、生成性检索）
7. 数字化使用对自传体记忆的影响（拍照效应、Google效应、认知卸载）

#### S1-执行心理学文献检索任务

## 触发条件
需要进行心理学领域文献检索，建立项目知识库并生成检索报告

## 执行方式
子代理执行

**子代理执行上下文**：
- **技能**：
  - `semantic-scholar-mcp`：用于Semantic Scholar文献检索
    - 使用方法：`mcporter call semantic-scholar.semantic_scholar_search query="关键词" limit=50`
  - `zotero`：用于Zotero文献管理和引用（可选）
- **密钥**：
  - `SEMANTIC_SCHOLAR_API_KEY`：Semantic Scholar API密钥（从~/.openclaw/.env读取）
- **任务要求**：
  - 按照步骤2的检索流程执行多轮检索
  - 收集奠基/重要/新近三类文献
  - 整理文献并添加labels字典
  - 生成符合规范的index.json
  - 撰写结构化检索报告
- **执行方案/步骤**：
  1. 接收主代理传递的检索主题和关键词
  2. 使用Semantic Scholar执行多轮检索
  3. 排除预印本、手稿等非期刊文献
  4. 按引用量分级（🔴奠基500+/🟡重要50-500/🔵一般<50）
  5. 添加labels字典（importance/type/jcr_quartile）
  6. 生成index.json和检索报告
  7. 返回结果给主代理

## 依赖
- 检索文献.md (general-scripts)：文献检索并更新项目知识库
- Semantic-Scholar-mcp 接口：用于学术文献检索，唯一允许的网络检索来源

## 输入
- 心理学研究主题
- 检索范围（时间范围、文献类型）
- 文献分级标准（奠基：引用量>500；重要：100-500；一般：<100）

## 步骤

### 步骤1：细化检索要求
- 检索陈述性记忆中的知识网络，明确研究主题的领域归属
- **扩展关键词**：提取核心关键词的同义词、下位词、相关概念
- **构建检索词组合**：使用多个不同的检索词组合覆盖主题全貌
- 梳理该领域的关键作者、经典研究、核心期刊
- 确定检索的纳入排除标准、时间范围、文献类型要求

### 步骤2：检索文献

**禁止规则**：
- 严禁使用Semantic Scholar之外的任何检索方式
- 严禁编造、杜撰文献信息或检索结果
- 检索出错必须如实报告错误类型和原因，不得隐瞒

**排除文献类型**：
- 预印本（preprint）
- 手稿（manuscript）
- 工作论文（working paper）

**期刊要求**：
- 仅保留JCR 1-4区期刊（后续筛选）

**检索流程**：
- **Semantic Scholar网络检索**：
  - 使用多个不同的检索词组合进行多轮检索
  - 🔴 **奠基文献检索**：引用量500+，全部保留
  - 🟡 **重要文献检索**：引用量50-500，全部保留
  - 🟢 **新近文献检索**：近3年实证研究，全部保留
- 合并检索结果并去重，按照影响因子、引用量、相关性排序
- 所有检索过程必须留痕，记录检索词、检索时间、返回数量

### 步骤3：整理文献

**排除非期刊文献**：
- 根据venue字段排除预印本、手稿、图书章节等

**排除内容无关文献**：
- 根据abstract字段排除与主题无关的文献等

**添加labels字典**：
- `importance`：🔴奠基（500+）/ 🟡重要（50-500）/ 🔵一般（<50）
- `type`：根据标题、摘要和期刊标题分类
  - 📖**综述**：标题明确包含review、meta-analysis、systematic review；或期刊标题含review、trends等
  - 💡**理论**：期刊标题含perspective、opinion、commentary、viewpoint等，或文章标题含theoretical、theory、viewpoint、commentary等理论关键词
  - 📊**实证**：摘要中包含被试/样本/方法/结果描述（关键词：participant, subject, sample, method, result, experiment, data, analysis等）
  - 📋**待分类**：无法明确归类为上述三类的文献
- `jcr_quartile`：Q1/Q2/Q3/Q4/待筛选（默认）

### 步骤4：统计文献
- 主题分布统计（适用于多个主题）
- 年份分布统计：>10年、5-10年、3-5年、3年内
- 文献类型分布：📊实证 / 📖综述 / 💡理论
- 重要性分布：🔴奠基 / 🟡重要 / 🔵一般
- 核心作者统计（发文数量）
- 核心期刊统计（载文数量）

### 步骤5：撰写检索报告

**检索报告结构**：

1. **检索概况**
   - 关键词
   - 搜索引擎
   - 搜索过程
   - 剔除标准
   - 文献最终数量

2. **文献统计**
   - 主题（适用于多个主题）
   - 年份：>10年、5-10年、3-5年、3年内
   - 类型：综述、实证、其他
   - 重要性分布：奠基/重要/一般
   - 主要作者
   - 主要期刊

3. **高引用文献（APA 7th格式）**
   - 按照引用数排序，列出前30篇

### 步骤6：管理与交付文档
- 按照「创建文件.md」创建检索报告文档（终稿/目录）
- 按照「控制版本.md」进行版本控制
- 返回本地路径、版本信息、云文档链接（如已上传）

- **询问用户是否上传云文档**，如需上传则按照「映射云文档.md」更新cloud_doc_mappings

#### S2-执行多主题并行文献检索任务

**触发条件**: 需要检索多个主题的文献（≥3个主题）

**执行方式**: 主代理协调 + 多个子代理并行执行（最多2个子代理同时运行）

**依赖**: 
- semantic-scholar-mcp 技能
- 工作记忆追踪系统
- S1脚本（单主题检索标准）

**输入**: 
- 主题列表（每个主题包含名称和检索关键词）
- 每主题文献分级标准（奠基/重要/新近）
- 并行度限制（默认最多2个子代理）

**步骤**:
1. **工作记忆初始化**
   - 在MEMORY.md工作记忆区创建子代理任务追踪表
   - 为每个主题分配子代理ID占位符
   - 设置状态为"⏳ 待执行"

2. **分批创建子代理**
   - 按并行度限制（2个）分批创建子代理
   - **向子代理传递完整上下文**：
     - API Key: `SEMANTIC_SCHOLAR_API_KEY`
     - 技能: `semantic-scholar-mcp` (使用 `mcporter call semantic-scholar.semantic_scholar_search`)
     - 检索标准: 奠基(最早+高引用,≤3篇)/重要(近20年引用500+)/新近(近3年实证,≤30篇)
     - 主题名称和检索关键词
     - 输出文件路径: `/tmp/topic{N}_{name}.json`
   - 每个子代理执行S1脚本检索一个主题
   - 更新工作记忆状态为"🔄 执行中"

3. **监控与调度**
   - 监听子代理完成事件
   - 每完成一个主题，立即创建下一个主题的子代理
   - 更新工作记忆状态和结果摘要

4. **结果整合**
   - 所有主题完成后，读取各主题索引文件
   - 去重、分级、整合到统一知识库
   - 生成综合检索报告

5. **工作记忆清理**
   - 标记所有子代理任务为"✅ 已完成"
   - 归档到事件记忆
   - 更新程序性记忆（如有新模式）

**输出**:
- 各主题独立索引文件（topic{N}_{name}.json）
- 整合后的知识库（index.json）
- 综合检索报告
- 工作记忆更新记录

### 条件-行动规则（If-Then Rules）

| 条件 | 行动 |
|------|------|
| 需要心理学领域文献检索（单主题） | 调用 S1 执行单主题文献检索任务，**使用 semantic-scholar-mcp 技能** |
| 需要心理学领域文献检索（多主题≥3） | 调用 S2 执行多主题并行文献检索任务，**最多2个子代理并行** |
| 检测到心理学概念不清 | 要求澄清概念定义，提供标准化定义建议 |
| 发现理论假设不可证伪 | 标记为不科学的理论，建议调整假设 |
| 实证支持不足 | 建议加强实证研究，提供研究设计建议 |
| 研究方法存在严重缺陷 | 指出方法学问题，提供改进方案 |
| **写入云文档** | **检查是否按映射云文档.md要求写入链接**|
