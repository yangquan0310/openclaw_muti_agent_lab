### 1.3 工作记忆使用规则

#### 初始化规则（计划阶段）
- 每次复杂任务开始前，主代理必须先加载此看板，检查是否有未完成任务
- 拆解子任务后，立即添加新行到「当前活跃任务看板」，状态为`active`
- 为每个任务分配唯一任务ID（格式：T001, T002, ...）

#### 执行规则（监控阶段）
- 创建子代理后，将子代理key和分配的任务填入「活跃子代理清单」
- 每完成一个子任务 → 更新任务状态为`completed`，结果摘要填入备注
- 当需要并行执行独立子任务时 → 使用子代理，状态改为`active`
- 子代理完成（收到通告）后，将任务状态改为`completed`，结果摘要填入备注
- 发现偏离计划超过 10% → 暂停并更新状态为`paused`，请求用户指示

#### 清理规则（调节阶段）
- 任务全部结束后，将任务状态改为`completed`
- 将本次任务看板中的历史记录移动到陈述性记忆区作为归档
- 清空工作记忆区的已完成任务
- 保留`active`状态的任务用于持续监控

#### 状态说明
- `active`: 子代理正在工作或待命（执行中）
- `paused`: 任务暂停，等待用户反馈
- `completed`: 任务已完成，子代理已归档

#### 一致性维护
- 确保「当前活跃任务看板」和「活跃子代理清单」状态一致
- 每日清理已归档超过7天的历史记录
- 每次任务结束后执行闭环自检清单

## 五、程序性记忆（Procedural Memory）

### 脚本（Scripts）

#### S1-执行文献综述任务
```
触发条件: 需要撰写文献综述
执行方式: 子代理执行
依赖: 
  - 检索文献.md (general-scripts)：文献检索并更新项目知识库
  - Semantic Scholar API：用于学术文献检索
  - Zotero API：用于文献管理和引用
输入: 研究主题、综述范围、文献要求
输出: 结构化文献综述文档（APA 7th格式）

步骤:
  1. 主题界定与范围确定
     - 明确综述的主题和研究领域
     - 确定文献检索的时间范围和语言范围
     - 明确纳入和排除标准

  2. 文献检索与收集
     - 使用Semantic Scholar等学术搜索引擎检索相关文献
     - 按照既定标准筛选高质量文献
     - 将检索到的文献保存到项目知识库中
     - 建立文献索引和元数据记录

  3. 文献阅读与摘要提取
     - 基于项目知识库系统阅读文献全文
     - 提取关键信息：研究目的、方法、结果、结论
     - 记录文献质量评价和操作化定义
     - 创建文献阅读笔记并存入知识库

  4. 文献分析与分类
     - 按照主题、方法、结论对文献进行分类
     - 识别研究趋势和理论脉络
     - 发现研究空白和争议点
     - 分析不同研究的操作化定义和测量方法

  5. 综述结构设计（问题导向）
     - 引言：介绍研究背景和综述重要性
     - 研究问题：梳理该领域大家关注的核心问题
     - 研究方法：分析大家如何研究这个问题，重点关注操作化定义和问卷测量方法
     - 研究结果：归纳发现的重要现象和规律
     - 讨论：解释这些现象的机制、理论贡献和研究局限
     - 未来研究方向：基于文献分析提出未来研究建议

  6. 综述撰写与整合
     - 按照问题导向结构撰写综述内容
     - 整合不同文献观点，构建理论脉络
     - 确保逻辑连贯，论证充分
     - 重点关注研究方法和操作化定义的比较分析

  7. 批判性分析与评述
     - 对现有研究的操作化定义进行批判性分析
     - 识别方法学局限和测量问题
     - 提出理论解释的替代方案
     - 总结研究范式的发展趋势

  8. 引用格式与参考文献整理
     - 采用APA 7th格式规范所有引用
     - 整理完整的参考文献列表
     - 验证引用信息的准确性和一致性
     - 确保所有引用都来自项目知识库中的真实文献
```
