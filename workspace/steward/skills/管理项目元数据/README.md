# 管理项目元数据

> 版本: 1.0.0  
> 维护者: 大管家  
> 创建时间: 2026-04-08

## 功能描述

本脚本用于创建新项目元数据，维护本地文档，以及维护本地文档与云文档（飞书/腾讯文档）的映射关系。

## 使用方法

1. **确定项目路径**：确认项目已存在于 `~/实验室仓库/项目文件/项目名/`
2. **准备元数据**：填写项目ID、标题、创建日期、描述、状态等信息
3. **配置云文档映射**（可选）：添加飞书或腾讯文档的映射关系
4. **写入元数据文件**：生成 `元数据.json` 文件
5. **验证**：检查JSON格式是否正确

## 文件结构

```
管理项目元数据/
├── 管理项目元数据.md    # 核心脚本（五要素SOP）
├── SKILL.md             # 给AI看的技能说明
└── README.md            # 给人类看的说明（本文件）
```

## 元数据字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `project_id` | string | 是 | 项目唯一标识，通常与文件夹名一致 |
| `title` | string | 是 | 项目标题，用于展示 |
| `created_date` | string | 是 | 创建日期，格式：YYYY-MM-DD |
| `status` | string | 是 | 项目状态：`active` 或 `completed` |
| `version` | string | 是 | 当前版本号，格式：v1, v2... |
| `description` | string | 否 | 项目描述，说明项目目标和内容 |
| `directories` | object | 否 | 项目目录结构记录 |
| `files` | array | 否 | 项目文件列表 |
| `tags` | array | 否 | 项目标签，用于分类和检索 |
| `tasks` | array | 否 | 项目任务记录 |
| `cloud_doc_mappings` | object | 否 | 云文档映射关系 |

## 云文档映射

支持飞书（feishu）和腾讯文档（tencent）两个平台：

```json
{
  "文档标题": {
    "local_path": "文档标题.md",
    "cloud": [
      {
        "platform": "feishu",
        "cloud_url": "https://...",
        "cloud_id": "doc-xxx",
        "created_at": "2026-04-08T10:00:00Z",
        "updated_at": "2026-04-08T10:00:00Z"
      }
    ]
  }
}
```

## 注意事项

- 项目路径必须已存在，否则提示错误
- 文件权限设置为644
- 时间戳使用ISO格式
