# 项目数据结构

> 本文件定义 manage-project 技能使用的完整数据结构

## 元数据整体结构

```json
{
  "project_id": "项目名称",
  "title": "项目标题",
  "created_date": "创建日期",
  "status": "active",
  "version": "v1",
  "description": "项目描述",
  "directories": {},
  "documents": [],
  "manuscripts": [],
  "notes": [],
  "reviews": [],
  "tags": [],
  "cloud_doc_mappings": {},
  "knowledge_base": {},
  "updated_at": "ISO时间戳"
}
```

## 字段详细说明

### 基础信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `project_id` | string | 是 | 项目唯一标识，与文件夹名一致 |
| `title` | string | 是 | 项目标题，用于展示 |
| `created_date` | string | 是 | 创建日期，格式：YYYY-MM-DD |
| `status` | string | 是 | 项目状态：`active` 或 `completed` |
| `version` | string | 是 | 当前版本号，格式：v1, v2... |
| `description` | string | 否 | 项目描述 |
| `updated_at` | string | 否 | 最后更新时间（ISO格式） |

### directories 结构

记录项目标准目录结构：

```json
{
  "文档": "文档/",
  "手稿": "手稿/",
  "知识库": "知识库/",
  "知识库/笔记": "知识库/笔记/",
  "知识库/综述": "知识库/综述/",
  "临时数据": "临时数据/",
  "临时数据/草稿": "临时数据/草稿/"
}
```

| 字段 | 值类型 | 说明 |
|------|--------|------|
| 文档 | str | 用户上传文档的目录路径 |
| 手稿 | str | 写作助手等撰写的最新md文档 |
| 知识库 | str | 项目专属知识库根目录 |
| 知识库/笔记 | str | 笔记输出目录 |
| 知识库/综述 | str | 综述输出目录 |
| 临时数据 | str | 临时文件、中间文件目录 |
| 临时数据/草稿 | str | 备份版本、中间文件目录 |

### documents 结构

记录用户上传的文档（docx/pdf等），不记录代理撰写的文件。

```json
[
  {
    "title": "文档标题",
    "version": "v1",
    "path": "文档/文档标题.docx",
    "type": "user_uploaded"
  }
]
```

| 字段 | 值类型 | 说明 |
|------|--------|------|
| title | str | 文档标题 |
| version | str | 文档版本号（如 v1、v2） |
| path | str | 文档相对路径 |
| type | str | 固定为 `user_uploaded` |

### manuscripts 结构

记录手稿目录中的代理撰写文档（.md文件）。

```json
[
  {
    "title": "手稿标题",
    "version": "v1",
    "path": "手稿/手稿标题.md",
    "type": "agent_written"
  }
]
```

| 字段 | 值类型 | 说明 |
|------|--------|------|
| title | str | 手稿标题 |
| version | str | 版本号 |
| path | str | 相对路径 |
| type | str | 固定为 `agent_written` |

### notes 结构

记录知识库/笔记目录中的笔记文件。

```json
[
  {
    "title": "笔记标题",
    "path": "知识库/笔记/笔记标题.json",
    "type": "note"
  }
]
```

### reviews 结构

记录知识库/综述目录中的综述文件。

```json
[
  {
    "title": "综述标题",
    "path": "知识库/综述/综述标题.md",
    "type": "review"
  }
]
```

### tags 结构

项目标签数组：

```json
[
  "AI降重",
  "提示工程",
  "学术写作"
]
```

### cloud_doc_mappings 结构

记录本地文档与云文档的映射关系：

```json
{
  "文档标题.md": {
    "local_path": "手稿/文档标题.md",
    "cloud": [
      {
        "platform": "feishu",
        "cloud_url": "https://...",
        "cloud_id": "doc-xxx",
        "created_at": "ISO时间戳",
        "updated_at": "ISO时间戳"
      }
    ]
  }
}
```

| 字段 | 值类型 | 说明 |
|------|--------|------|
| local_path | str | 本地文档路径 |
| platform | str | 云文档平台：`feishu` 或 `tencent` |
| cloud_url | str | 云文档URL |
| cloud_id | str | 云文档ID |
| created_at | str | 云文档创建时间（ISO格式） |
| updated_at | str | 云文档更新时间（ISO格式） |

### knowledge_base 结构

记录知识库索引信息：

```json
{
  "index_file": "知识库/index.json",
  "description": "项目知识库索引",
  "created_at": "2026-04-01",
  "updated_at": "2026-04-16"
}
```

## 文件类型映射

系统根据文件扩展名自动识别文件类型：

| 类型 | 扩展名 |
|------|--------|
| user_uploaded | .docx, .pdf, .txt, .doc, .xlsx, .pptx |
| agent_written | .md |
| backup | .bak, .old, .backup |
| intermediate | .tmp, .draft, .intermediate |

## 目录结构规范

项目标准目录结构：

```
project/
├── 文档/              # 用户上传文档
├── 手稿/              # 写作助手、审稿助手等撰写的最新md文档
├── 知识库/
│   ├── 笔记/          # 笔记输出
│   └── 综述/          # 综述输出
└── 临时数据/
    └── 草稿/          # 备份版本、中间文件
```

## 整理工作流

`organize()` 方法的整理规则：

1. **中间文件** → `临时数据/`
   - 备份文件(.bak/.old/.backup)
   - 中间文件(.tmp/.draft/.intermediate)
   - 其他未分类文件

2. **综述输出** → `知识库/综述/`
   - 文件名包含"综述"或"review"的文件

3. **笔记输出** → `知识库/笔记/`
   - 知识库根目录下的其他文件

4. **用户上传** → `文档/`
   - docx/pdf/txt/doc/xlsx/pptx 等格式

5. **代理撰写** → `手稿/`
   - .md 格式文件

6. **备份版本** → `临时数据/草稿/`
   - 旧"草稿"目录中的文件
   - 旧"终稿"目录中的非md文件
