## 根因调查过程

### 问题现象
用户发现所有项目的 `元数据.json` 中 `markdown` 字段的 `cloud` 数组均为空 `[]`。

### 调查步骤（遵循"先查根因再修复"规则）

**Step 1: 读代码**
- 检查维护脚本 `维护所有项目元数据.py`
- 发现使用旧字段名 `cloud_doc_mappings`，与 SKILL.md 标准结构 `markdown` 不匹配

**Step 2: 定位问题**
- 脚本重建元数据时未正确读取 `markdown` 字段
- 导致 `cloud` 数据丢失

**Step 3: 确认根因**
- 确认是字段名不匹配导致的数据丢失
- 非故意删除，是脚本bug

### 修复过程

**第一次尝试（错误）**：
- 直接修改旧脚本 `维护所有项目元数据.py`
- 用户指出：应该使用 manage-project 技能的维护脚本

**第二次尝试（正确）**：
- 读取 manage-project 技能的 `Maintainer.py`
- 发现 `update_metadata()` 方法在扫描手稿目录时**直接覆盖** `markdown` 字段
- 将 `cloud` 设为空列表 `[]`

**最终修复**：
- 修改 `Maintainer.py` 第900行附近代码
- 保留现有 `cloud` 数据，只更新 `local_path`

```python
# 修复前（错误）
markdown[filename] = {
    "local_path": f"手稿/{filename}",
    "cloud": []  # ← 清空云文档链接
}

# 修复后（正确）
existing_cloud = []
if "markdown" in self.metadata and filename in self.metadata["markdown"]:
    existing_entry = self.metadata["markdown"][filename]
    if "cloud" in existing_entry:
        existing_cloud = existing_entry["cloud"]

markdown[filename] = {
    "local_path": f"手稿/{filename}",
    "cloud": existing_cloud  # ← 保留云文档链接
}
```
