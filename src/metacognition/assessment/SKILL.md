# Assessment — 任务复杂度评估与 Plan 决策

> 元认知子模块 - 评估阶段
> **Plugin asks, Agent decides, User confirms**
> 插件不替 Agent 判断，Agent 自行评估，用户最终确认

---

## 注入上下文

本 skill 在 **`before_prompt_build`** 触发时注入，**仅在当前 runId 尚无 task JSON 时注入**。此时插件已完成以下操作：

| 时机 | 插件已完成的操作 | 说明 |
|------|-----------------|------|
| 用户提问后 | 检查 `task:{runId}` 是否存在 | 若不存在，注入本 skill |

**当前状态**：无 task，Agent 需要自行判断是否需要制定 Plan。

---

## Agent 职责

### 职责：评估任务复杂度并决定是否需要 Plan

**你需要做的**（决策层）：

1. **阅读用户当前 prompt**
   - 理解用户意图和需求

2. **评估任务复杂度**

   | 类型 | 判定标准 | 示例 | Agent 行动 |
   |------|---------|------|-----------|
   | **简单任务** | 单步骤、无需工具、即时可回答 | 查询、翻译、计算、总结、闲聊、天气、时间 | 直接回答用户，**不创建 Plan** |
   | **中等任务** | 2-3 个步骤、可能需工具、有明确产出 | 写一段代码、分析一个文件、生成一张图 | 向用户说明复杂度，**询问是否需要 Plan** |
   | **复杂任务** | 多步骤、需要多个工具、涉及子任务、需要验证 | 开发一个功能、重构项目、写论文、数据分析报告 | 向用户说明复杂度，**询问是否需要 Plan** |

3. **向用户汇报评估结果**

   - **简单任务**：直接回答，无需额外说明
   - **中等/复杂任务**：
     ```markdown
     📋 **任务评估**

     这个任务涉及 [N] 个步骤，预计需要 [工具/操作]。
     建议制定一个执行计划，明确各阶段目标和产出。

     是否需要我制定 Plan？（回复"是"或"否"）
     ```

4. **用户确认后输出标记**

   - 如果用户确认需要 Plan → 在你的回复末尾添加 `[NEED_PLAN]` 标记
   - 如果用户拒绝 → 直接按简单方式处理，**不要添加标记**

---

## 决策检查点

每次收到本 skill 注入时，请确认：

- [ ] 用户任务是否可以在 1-2 轮对话内完成？（是 → 简单任务）
- [ ] 任务是否需要调用外部工具或读写文件？（是 → 可能需要 Plan）
- [ ] 任务是否有多个明确的子目标？（是 → 建议 Plan）
- [ ] 是否已向用户说明复杂度并询问是否需要 Plan？（中等/复杂任务必做）
- [ ] 用户确认后是否已添加 `[NEED_PLAN]` 标记？

## 状态流转

```
用户提问
    │
    ▼
注入 assessment skill（无 task）
    │
    ▼
Agent 评估任务复杂度
    │
    ├─ 简单 ──→ 直接回答用户（结束，不创建 task）
    │
    └─ 中等/复杂 ──→ 询问用户："是否需要制定 Plan？"
              │
              ├─ 用户确认 "是" ──→ 回复末尾添加 [NEED_PLAN]
              │                     │
              │                     ▼
              │              插件检测到标记，创建 task，注入 planning skill
              │                     │
              │                     ▼
              │              Agent 制定 Plan → 用户确认 → active → 执行
              │
              └─ 用户拒绝 "否" ──→ 直接回答用户（结束，不创建 task）
```
