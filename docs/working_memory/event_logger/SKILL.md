# event_logger

> 工作记忆 - 事件日志记录
> 记录元认知过程到 events 文件夹

---

## 概述

事件日志记录器负责将元认知过程中的关键事件记录到 `~/.openclaw/workspace/{agentId}/events/` 文件夹，为后续的日记生成和自我更新提供素材。

---

## 存储规范

### 存储位置

- **事件目录**: `~/.openclaw/workspace/{agentId}/events/YYYY-MM-DD/`
- **日记目录**: `~/.openclaw/workspace/{agentId}/diary/`

### 文件命名

**事件日志**:
- **格式**: `HH-MM-SS-事件描述.md`
- **示例**: `14-30-00-创建文献检索子代理.md`
- **完整路径**: `events/2026-04-17/14-30-00-创建文献检索子代理.md`

**发展日记**:
- **格式**: `YYYY-MM-DD.md`
- **示例**: `2026-04-17.md`
- **完整路径**: `diary/2026-04-17.md`

---

## 核心功能

### 1. 创建事件日志

**触发条件**:
| 事件类型 | 触发阶段 | 示例 |
|----------|----------|------|
| 子代理创建 | 计划 | 创建文献检索子代理 |
| 偏差检测 | 监控 | 检测到进度偏差 |
| 调节执行 | 调节 | 调整检索策略 |
| 子代理完成 | 完成 | 文献检索完成 |
| 子代理终止 | 终止 | 终止分析子代理 |
| 计划修正 | 调节 | 修正分析范围 |

**创建流程**:
```
触发事件
    ↓
[步骤1] 确定目录和文件名
    └── 目录: events/YYYY-MM-DD/
    └── 格式: HH-MM-SS-事件描述.md
    ↓
[步骤2] 复制模板
    └── 从 regulation/templates/事件日志模板.md 复制
    ↓
[步骤3] 填写内容
    ├── 基本信息
    ├── 事件描述
    ├── 元认知阶段
    ├── 执行结果
    └── 经验总结
    ↓
[步骤4] 添加日记标记
    ├── 同化标记
    └── 顺应标记
    ↓
[步骤5] 保存到 events/YYYY-MM-DD/ 目录
```

### 2. 读取事件日志

**供日记生成使用**:
```
每日 01:00 定时任务
    ↓
[步骤1] 读取前一天所有事件日志
    └── 扫描 events/YYYY-MM-DD/*.md
    ↓
[步骤2] 按时间排序
    ↓
[步骤3] 提取关键信息
    ├── 事件类型
    ├── 执行结果
    └── 经验总结
    ↓
[步骤4] 生成发展日记
    └── 保存到 diary/YYYY-MM-DD.md
```

### 3. 管理事件日志

**保留策略**:
| 类型 | 保留时间 | 处理方式 |
|------|----------|----------|
| 事件日志 | 30 天 | 自动归档到长期存储 |
| 发展日记 | 永久 | 保留在 diary/ 目录 |

## 事件日志内容规范

### 必须包含的章节

1. **基本信息** - 时间、类型、项目、子代理
2. **事件描述** - 背景、触发条件、详情
3. **元认知阶段** - 计划/监控/调节相关信息
4. **执行结果** - 结果状态、实际结果、与预期对比
5. **经验总结** - 成功经验、新发现、可复用模式
6. **日记标记** - 同化标记、顺应标记

### 日记标记规范

**同化标记** (纳入现有认知):
- `[ ] 可纳入现有流程`
- `[ ] 可复用的成功模式`
- `[ ] 标准操作程序更新`

**顺应标记** (调整认知结构):
- `[ ] 发现认知盲区`
- `[ ] 需要调整策略`
- `[ ] 需要更新技能/知识`

## 与 assimilation_accommodation 的关系

```
event_logger (记录事件)
    ├── 创建 events/YYYY-MM-DD/HH-MM-SS-事件.md
    └── 添加日记标记
            ↓ (每日 01:00)
    diary_reader (读取事件)
        └── 读取所有事件日志
                ↓
    diary (生成日记)
        └── 创建 diary/YYYY-MM-DD.md
                ↓ (每日 02:00)
    自我更新 (同化/顺应)
        ├── core_self_update
        ├── identity_update
        ├── belief_style_update
        ├── self_identity_update
        └── personal_skills_update
```
