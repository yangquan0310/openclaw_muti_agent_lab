# personal_skills_update

> 同化与顺应 - 个人技能更新
> 更新 Agent 个人技能文件和技能索引

---

## 概述

个人技能更新负责维护和更新 Agent 的个人技能文件，包括技能索引、技能文档和技能脚本。当发展日记识别到技能相关的同化或顺应需求时，执行相应的更新操作。

---

## 更新范围

### 1. 技能索引更新

**更新文件**: `~/.openclaw/workspace/{agentId}/skills/README.md`

**更新内容**:
- 新增技能条目
- 更新技能描述
- 修改技能路径
- 删除过时技能

### 2. 技能文档更新

**更新文件**: `~/.openclaw/workspace/{agentId}/skills/{skillName}/SKILL.md`

**更新内容**:
- 创建新技能文档
- 更新现有技能内容
- 重构技能结构
- 删除过时技能文档

### 3. 技能脚本更新

**更新文件**: `~/.openclaw/workspace/{agentId}/skills/{skillName}/scripts/`

**更新内容**:
- 添加新脚本
- 修改现有脚本
- 删除无用脚本

---

## 同化更新 (纳入现有技能)

**触发条件**:
- 发现现有技能的新应用场景
- 总结出现有技能的最佳实践
- 识别出可复用的成功模式

**更新操作**:
```markdown
1. 更新技能描述，添加新场景
2. 在 SKILL.md 中添加最佳实践章节
3. 添加条件-行动规则到程序性记忆
4. 更新技能使用示例
```

---

## 顺应更新 (调整技能结构)

**触发条件**:
- 发现现有技能无法满足新需求
- 识别出技能盲区
- 需要创建全新技能

**更新操作**:
```markdown
1. 创建新技能目录和 SKILL.md
2. 更新 skills/README.md 索引
3. 添加新技能到个人技能列表
4. 记录技能创建原因和使用场景
```

---

## 更新流程

```
从发展日记接收更新需求
    ↓
[步骤1] 分析更新类型
    ├── 同化 → 更新现有技能
    └── 顺应 → 创建/重构技能
    ↓
[步骤2] 执行更新
    ├── 更新 skills/README.md
    ├── 更新/创建 SKILL.md
    └── 更新/创建脚本文件
    ↓
[步骤3] 验证更新
    ├── 检查文件完整性
    ├── 验证链接有效性
    └── 确认格式规范性
    ↓
[步骤4] 记录更新日志
    └── 在 MEMORY.md 事件记忆中记录
```
