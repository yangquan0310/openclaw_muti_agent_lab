## 对象

### Skills（技能对象）

**说明**：Skills 对象维护 Agent 的个人技能索引和技能文档，对应 `skills/` 目录下的内容。

**属性**：

| 属性 | 类型 | 说明 |
|------|------|------|
| `skillIndex` | object[] | 技能索引 `{ name, path, version, status }` |
| `templates` | object | 技能文档模板 |
| `deprecated` | string[] | 已淘汰技能列表 |
| `version` | string | 技能体系版本号 |

**状态变换**：

```
稳定态
    ↓ Diary 触发技能变化信号
分析中（判断同化或顺应）
    ├── 同化路径（现有技能细化）
    │   ↓
    更新中（调用 update()）
    │   ↓
    验证中（validate）
    └── 顺应路径（新技能创建）
        ↓
        创建中（调用 create()）
        ↓
        索引更新中（更新 skillIndex）
    ↓
    完成
返回稳定态
```

## 工作流（修改 Skills 对象的方法）

### 工作流1：同化更新（现有技能细化）

**说明**：通过本工作流修改 Skills 对象中现有技能的定义。

**触发条件**：
- 发现现有技能的新应用场景
- 总结出现有技能的最佳实践
- 识别出可复用的成功模式

**步骤**：
1. **定位技能**（读取 `Skills.skillIndex`）
   - 找到需要更新的技能条目

2. **更新技能定义**（修改技能 SKILL.md）
   - 更新技能描述，添加新场景
   - 在 SKILL.md 中添加最佳实践章节
   - 添加条件-行动规则到程序性记忆
   - 更新技能使用示例

3. **更新 Skills 对象**（修改 `Skills.skillIndex`）
   - 更新版本号
   - 更新修改时间

4. **验证**（Skills.validate()）
   - 检查文件完整性
   - 验证链接有效性
   - 确认格式规范性

5. **记录更新历史**

---

### 工作流2：顺应更新（新技能创建）

**说明**：通过本工作流在 Skills 对象中创建新技能。

**触发条件**：
- 发现现有技能无法满足新需求
- 识别出技能盲区
- 需要创建全新技能

**步骤**：
1. **定义新技能**
   - 技能名称
   - 功能描述
   - 使用场景

2. **创建技能文档**（修改 `Skills.skillIndex` + 新建文件）
   - 创建新技能目录和 SKILL.md
   - 从 `Skills.templates` 复制模板

3. **更新索引**（修改 `Skills.skillIndex`）
   - 更新 `skills/README.md` 索引
   - 添加新技能到个人技能列表

4. **记录创建原因**
   - 在技能文档中记录创建原因和使用场景

5. **验证**（Skills.validate()）

---

### 工作流3：技能淘汰与迁移

**说明**：通过本工作流修改 Skills 对象，淘汰过时技能。

**步骤**：
1. **标记淘汰**
   - 将技能加入 `Skills.deprecated`
   - 在 skillIndex 中标记状态为 `deprecated`

2. **依赖迁移**
   - 更新依赖关系到新版本或替代技能

3. **归档或删除**
   - 轻度淘汰：保留文件，标记 deprecated
   - 重度淘汰：移动至 `archive/` 目录
