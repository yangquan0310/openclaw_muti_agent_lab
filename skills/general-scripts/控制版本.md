# 控制版本SOP

## 触发条件
需要管理文档版本，备份旧版本，更新当前版本号

## 执行方式
当前会话主代理执行

## 输入
- 项目名称：目标项目
- 文件路径：相对于项目根目录的路径
- 版本升级方式：`major` / `minor` / `patch`（默认 patch）

## 输出
- 旧版本已备份到 `草稿/versions/`
- 当前文件版本号已更新
- 版本信息已记录到项目元数据

## 依赖
- 写入项目元数据.md：更新项目元数据中的版本信息

## 步骤

1. **确认版本信息**
   - 确认项目和文件路径
   - 确认版本升级方式

2. **读取当前版本**
   - 从项目元数据读取当前版本号
   - 如果没有则初始化为 v1.0.0

3. **计算新版本号**
   - `major`: 主版本+1，次版本=0，修订=0 (v1.0.0 → v2.0.0)
   - `minor`: 主版本不变，次版本+1，修订=0 (v1.0.0 → v1.1.0)
   - `patch`: 主版本不变，次版本不变，修订+1 (v1.0.0 → v1.0.1)

4. **备份旧版本**
   - 创建备份目录 `草稿/{filename}/`（如果不存在），`{filename}` 是当前文件去掉扩展名
   - 复制当前文件到备份目录
   - 命名格式：`{filename}_v{old_version}.md`

5. **更新当前版本信息**
   - 更新项目元数据中的版本号
   - 如果文档开头有版本记录，同步更新

6. **验证**
   - 检查备份文件是否存在
   - 检查版本号是否正确更新

7. **记录到工作日志**
   - 记录升级时间、项目、文件、版本变化
