# 管理知识库SOP

## 触发条件
需要将检索到的文献存入项目知识库的 `index.json` 中，包括添加文献、更新文献、删除文献

## 执行方式
当前会话主代理执行

## 输入
- 项目名称：目标项目（如 `2026-04-01_数字化存储与自传体记忆`）
- 文献信息：文献列表，包含标题、作者、年份、DOI、摘要、链接等字段
- 操作类型：`add`（添加）/ `update`（更新）/ `delete`（删除）

## 输出
- 更新后的 `index.json` 文件保存在 `~/实验室仓库/项目文件/{项目名}/知识库/index.json`
- 输出操作结果统计（添加/更新/删除数量）

## 依赖
- 目标项目已创建，目录结构完整
- 文献信息格式完整（至少包含标题和作者）

## index.json 结构

```json
{
  "version": 1,
  "project": "项目名称",
  "created_at": "ISO时间戳",
  "updated_at": "ISO时间戳",
  "total_count": 0,
  "entries": [
    {
      "id": "自增ID",
      "title": "文献标题",
      "authors": "作者列表，逗号分隔",
      "year": "出版年份",
      "doi": "DOI 号",
      "abstract": "摘要",
      "url": "文献链接",
      "venue": "发表期刊/会议",
      "citationCount": "引用量",
      "tags": ["标签1", "标签2"],
      "created_at": "添加时间戳",
      "updated_at": "最后更新时间戳"
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `version` | integer | 是 | index 结构版本，当前为 1 |
| `project` | string | 是 | 项目名称 |
| `created_at` | string | 是 | 知识库创建时间 |
| `updated_at` | string | 是 | 最后更新时间 |
| `total_count` | integer | 是 | 总文献条数 |
| `entries` | array | 是 | 文献条目数组 |
| `entries[].id` | integer | 是 | 自增文献ID |
| `entries[].title` | string | 是 | 文献标题 |
| `entries[].authors` | string | 是 | 作者列表 |
| `entries[].year` | integer | 否 | 出版年份 |
| `entries[].doi` | string | 否 | DOI 号 |
| `entries[].abstract` | string | 否 | 摘要 |
| `entries[].url` | string | 否 | 文献链接 |
| `entries[].venue` | string | 否 | 发表期刊/会议 |
| `entries[].citationCount` | integer | 否 | 引用量 |
| `entries[].tags` | array | 否 | 标签列表 |
| `entries[].created_at` | string | 是 | 添加时间 |
| `entries[].updated_at` | string | 是 | 最后更新时间 |

## 步骤

1. **确认项目信息**
   - 获取项目名称
   - 定位项目目录：`~/实验室仓库/项目文件/{项目名}/知识库/`
   - 如果 `index.json` 不存在，按照上述结构创建空文件（`total_count: 0, entries: []`）

2. **读取现有 index.json**
   - 加载现有文献索引
   - 验证 JSON 格式是否正确

3. **执行操作**
   - **add**: 检查文献是否已存在（通过 DOI 查重），不存在则添加到 entries，分配自增ID，添加时间戳
   - **update**: 根据 ID 或 DOI 更新文献信息，更新 `updated_at` 时间戳
   - **delete**: 根据 ID 或 DOI 删除文献，重新编号 ID
   - 更新根级别的 `updated_at` 和 `total_count`

4. **写入更新后的 index.json**
   - 使用格式化 JSON 输出（便于阅读和版本控制）
   - 保存到原位置

5. **验证结果**
   - 检查文件是否成功保存
   - 统计操作结果：新增 N 条，更新 M 条，删除 K 条

6. **记录到工作日志**
   - 记录操作时间、项目名称、操作统计
