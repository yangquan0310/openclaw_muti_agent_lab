# 创建新项目SOP

## 触发条件
需要创建一个新的研究项目，初始化项目目录结构和基础文件

## 执行方式
当前会话主代理执行

## 输入
- 项目名称：`YYYY-MM-DD_项目名` 格式
- 项目标题：项目中文标题
- 项目描述：项目简要描述
- 创建日期：YYYY-MM-DD（默认今天）

## 输出
- 创建完成的项目根目录路径
- 初始化的 `元数据.json` 和 `README.md`

## 依赖
- `写入项目元数据.md`：创建并初始化项目元数据文件

## 步骤

1. **确认项目信息**
   - 确认项目名称格式正确（YYYY-MM-DD_项目名）
   - 确认项目标题和描述
   - 确认创建日期

2. **创建项目根目录**
   - 路径：`~/实验室仓库/项目文件/{项目名称}/`
   - 如果已存在则提示错误

3. **创建标准子目录**
   - `文档/`：用户上传的原始文档
   - `草稿/`：论文草稿
   - `终稿/`：最终版本
   - `知识库/`：项目专属知识库
   - 创建空的 `index.json` 知识库索引

4. **创建元数据.json**
   - 调用 `写入项目元数据.md` 脚本创建初始化元数据
   - 包含项目基本信息，空的 `cloud_doc_mappings` 数组

5. **创建 README.md**
   - 写入项目基本说明
   - 包含项目名称、描述、创建日期、目录结构

6. **验证**
   - 检查所有目录和文件是否创建成功
   - 验证 `元数据.json` JSON格式正确

7. **记录到工作日志**
   - 记录创建时间、项目名称、路径
