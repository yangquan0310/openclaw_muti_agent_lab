# 管理项目元数据SOP

## 触发条件
需要创建新项目元数据，或维护本地文件与云文档（飞书/腾讯文档）的映射关系

## 执行方式
当前会话主代理执行

## 依赖
- 目标项目已创建（对于映射云文档）

## 输入
### 创建元数据时：
- 项目ID：项目名称
- 项目标题：项目标题
- 创建日期：创建日期（YYYY-MM-DD）
- 项目描述：项目描述
- 状态：active / completed

### 映射云文档时：
- 项目名称：目标项目
- 本地路径：本地文件相对于项目根目录的路径
- 云文档URL：云文档的访问链接
- 云文档ID：云文档的ID（如果有）
- 平台：`feishu` / `tencent`

## 元数据中的映射结构

在项目 `元数据.json` 中，`cloud_doc_mappings` 是一个数组，每个映射项格式：

```json
{
  "cloud_doc_mappings": [
    {
      "local_path": "草稿/论文草稿.md",
      "cloud_url": "https://...",
      "cloud_id": "doc-xxx",
      "platform": "feishu",
      "created_at": "ISO时间戳",
      "updated_at": "ISO时间戳"
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `local_path` | string | 是 | 本地文件相对项目根目录的路径 |
| `cloud_url` | string | 是 | 云文档访问URL |
| `cloud_id` | string | 否 | 云文档ID |
| `platform` | string | 是 | 平台：`feishu` 或 `tencent` |
| `created_at` | string | 是 | 创建映射时间 |
| `updated_at` | string | 是 | 最后更新时间 |

## 步骤

### 创建项目元数据

1. **确定项目路径**
   - 项目路径：`~/实验室仓库/项目文件/YYYY-MM-DD_项目名/`
   - 检查路径是否存在，如果不存在则提示错误

2. **初始化元数据结构**
   ```json
   {
     "project_id": "项目名称",
     "title": "项目标题",
     "created_date": "创建日期",
     "status": "active",
     "version": "v1",
     "description": "项目描述",
     "directories": {},
     "files": [],
     "tags": [],
     "tasks": [],
     "cloud_doc_mappings": []
   }
   ```

3. **写入元数据.json**
   - 写入到项目根目录
   - 设置正确文件权限：644

4. **验证创建**
   - 检查文件是否存在
   - 验证JSON格式是否正确

### 映射云文档

1. **确认映射信息**
   - 确认项目名称
   - 确认本地文件路径
   - 确认云文档URL和平台
   - 确认云文档所有权已转移给用户

2. **读取项目元数据**
   - 加载 `~/实验室仓库/项目文件/{项目名称}/元数据.json`
   - 确认结构正确

3. **更新映射关系**
   - 在 `cloud_doc_mappings` 数组中添加/更新映射项
   - 映射项包含：`local_path`, `cloud_url`, `cloud_id`, `platform`, `created_at`, `updated_at`
   - 如果已存在相同 `local_path`，覆盖旧映射

4. **写入更新后的元数据**
   - 格式化JSON保存
   - 验证格式正确

5. **验证**
   - 检查映射是否已添加
   - 检查JSON格式是否正确

6. **记录到工作日志**
   - 记录映射更新时间、项目、映射信息

## 输出
- 创建完成的 `元数据.json` 文件路径（创建时）
- 更新后的项目元数据.json（映射时）
- 映射关系已保存
