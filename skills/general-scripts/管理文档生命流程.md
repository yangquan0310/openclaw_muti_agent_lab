# 文档全流程管理脚本

## 总体描述
本脚本整合了文档管理的完整生命周期，涵盖从本地文件创建到云文档同步、版本控制的全部流程。

## 核心功能
1. **本地文件操作**：创建、修改本地Markdown文件
2. **云文档同步**：智能上传到飞书/腾讯云文档
3. **映射管理**：维护本地文件与云文档的映射关系
4. **版本控制**：规范的版本管理和历史回溯

## 触发条件
**主触发条件**：需要进行文档相关的创建、修改、同步、版本控制等操作

**具体触发场景**：
- 需要创建新的文档或项目文件
- 需要修改现有文档内容
- 需要将文档同步到云平台（飞书/腾讯）
- 需要更新文档版本
- 需要管理云文档映射关系

## 执行方式
**智能决策执行**：根据具体场景自动选择执行路径

## 依赖
- [技能文件夹]/[技能]/SKILL.md

## 输入
- 文档内容或修改内容
- 目标平台（本地/飞书/腾讯）
- 项目信息
- 版本要求

## 输出
- 本地文件路径（如创建/修改）
- 云文档链接（如上传/同步）
- 更新的元数据.json
- 版本信息

## 详细步骤

### 1. 本地文件创建流程
**输入**：内容、目标目录、文件名
**输出**：创建的Markdown文件路径

**步骤**：
- 步骤1：确定项目信息
  - 项目名: [项目名称]
  - 文档标题: [文档标题]
- 步骤2：版本控制准备
  - 检查是否有同名文件
  - 如有，执行版本控制流程
- 步骤3：生成Markdown源文件
  - 按规范命名：YYYY-MM-DD_描述.md
  - 添加标准头部信息
- 步骤4：保存到指定目录
  - 路径: ~/实验室仓库/项目文件/[项目名]/终稿/[标题].md
- 步骤5：验证文件创建
  - 检查文件存在性
  - 检查文件权限
- 步骤6：记录到工作日志

### 2. 本地文件修改流程
**输入**：文件路径、修改内容、目标云平台
**输出**：修改后的文件路径

**步骤**：
- 步骤1：读取原文件内容
  - 使用 read() 读取完整内容
- 步骤2：执行修改
  - 使用 edit() 进行精确替换
  - 或使用 write() 完全重写
- 步骤3：验证修改结果
  - 检查修改是否正确
- 步骤4：判断云文档同步需求
  - 条件判断：
    - 需要同步到飞书云文档：执行飞书修改流程
    - 需要同步到腾讯云文档：执行腾讯修改流程
    - 需要同时同步到两者：先飞书后腾讯
- 步骤5：记录到工作日志

### 3. 云文档智能上传流程
**输入**：本地文件、项目元数据
**输出**：云文档链接、更新的元数据

**步骤**：
- 步骤1：检查项目元数据
  - 读取项目的元数据.json
  - 检查 cloud_doc_mappings 字段是否已有该文件的云文档链接
- 步骤2：决策分支
  - **如果没有云文档链接**：
    - 询问用户选择云平台（飞书/腾讯/两者）
    - 根据选择创建新的云文档
    - 上传内容到云文档
    - 管理云文档映射
  - **如果已有云文档链接**：
    - 直接上传内容到现有云文档
    - 更新同步时间
- 步骤3：云文档映射管理
  - 检查元数据.json中的cloud_doc_mappings
  - 添加或更新映射条目
  - 字段包括：本地文件名、飞书云文档（数组）、腾讯云文档（数组）、状态、同步时间、所有者等
- 步骤4：记录到工作日志

### 4. 云文档创建流程
**输入**：内容、目标平台、文档标题
**输出**：云文档链接、文档token

**步骤**：
- 步骤1：条件判断
  - 需要创建飞书云文档：执行飞书创建流程
  - 需要创建腾讯云文档：执行腾讯创建流程
  - 需要同时创建两者：先飞书后腾讯
- 步骤2：飞书云文档创建流程
  - 创建云文档（使用 feishu_doc.create()）
  - 指定 owner_open_id: "ou_25cf20a1973aecc51f73d8e2800d7f7e"
  - 写入内容（使用 feishu_doc.write() 或 append()）
  - 验证所有权（确认杨权为文档所有者）
- 步骤3：腾讯云文档创建流程
  - 创建云文档（使用 mcporter call tencent-docs.create_smartcanvas_by_mdx）
  - 指定 title 和 content_format: "markdown"
  - 写入内容（若内容较长，使用分段上传）
  - 验证创建成功
- 步骤4：共享步骤
  - 分享链接
  - 记录到工作日志

### 5. 云文档修改流程
**输入**：文档token或URL、修改内容、目标平台
**输出**：修改后的云文档链接

**步骤**：
- 步骤1：条件判断
  - 需要修改飞书云文档：执行飞书修改流程
  - 需要修改腾讯云文档：执行腾讯修改流程
  - 需要同时修改两者：先飞书后腾讯
- 步骤2：飞书云文档修改流程
  - 从飞书云文档获取最新内容（使用 feishu_doc.read()）
  - 更新本地Markdown文件
  - 理解用户修改逻辑
  - 在此基础上修改（如有必要）
  - 更新飞书云文档
- 步骤3：腾讯云文档修改流程
  - 从腾讯云文档获取最新内容（使用 mcporter call tencent-docs.smartcanvas.read）
  - 更新本地Markdown文件
  - 理解用户修改逻辑
  - 在此基础上修改（如有必要）
  - 更新腾讯云文档（使用 mcporter call tencent-docs.smartcanvas.edit）
- 步骤4：共享步骤
  - 记录到工作日志

### 6. 云文档映射管理流程
**输入**：云文档信息、本地文件信息、项目路径
**输出**：更新的元数据.json

**步骤**：
- 步骤1：读取现有云文档映射
  - 路径: 项目/元数据.json
  - 字段: cloud_doc_mappings
- 步骤2：添加或更新映射条目
  - 键为本地文件名（如：文献综述.md）
  - local_file: 路径、文件名、类型、版本、创建日期
  - 飞书云文档: 数组格式，支持多个云文档对应一个本地文档
  - 腾讯云文档: 数组格式
  - notes: 映射条目备注
- 步骤3：验证映射完整性
  - 检查本地文件存在性
  - 验证云文档可访问性
- 步骤4：保存映射
  - 更新元数据.json的cloud_doc_mappings字段
  - 更新last_updated字段
- 步骤5：记录到工作日志

### 7. 版本控制流程
**输入**：目标文档、新内容、项目信息
**输出**：更新后的文档路径、版本信息

**步骤**：
- 步骤1：确定目标文档信息
  - 项目名: [项目名称]
  - 标题: [文档标题]
  - 检查文件位置: ~/实验室仓库/项目文件/[项目名]/终稿/[标题].md
- 步骤2：检查当前版本号
  - 方式一: 读取项目元数据.json的version字段
  - 方式二: 检查草稿文件夹中的历史版本编号
  - 默认: 如果没有历史版本，当前版本为v1
- 步骤3：创建草稿文件夹结构
  - 路径: ~/实验室仓库/项目文件/[项目名]/草稿/[标题]文件头/
  - 如果目录不存在则创建
- 步骤4：备份旧版本到草稿文件夹
  - 读取终稿文件夹的当前内容
  - 保存为: ~/实验室仓库/项目文件/[项目名]/草稿/[标题]文件头/[标题]v[当前版本号].md
- 步骤5：更新终稿文件夹的最新版本
  - 将新内容写入: ~/实验室仓库/项目文件/[项目名]/终稿/[标题].md
- 步骤6：更新元数据.json中的版本信息
  - 版本号递增: v1 → v2 → v3...
  - 更新last_updated时间戳
  - 记录version_history数组（可选）
- 步骤7：同步终稿到云文档
  - 检查云文档映射
  - 如需要，上传新内容到云文档
  - 更新同步时间
- 步骤8：记录到工作日志
