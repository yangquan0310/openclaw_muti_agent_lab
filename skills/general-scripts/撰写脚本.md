# 撰写脚本SOP

## 触发条件
需要创建新的操作脚本，按照**触发条件、执行方式、输入、输出、依赖、步骤**六要素格式撰写

## 执行方式
当前会话执行，根据用户提供的脚本功能描述撰写完整SOP

## 输入
- 脚本名称：脚本的名称
- 脚本功能描述：脚本要完成什么任务
- 触发场景：什么时候需要触发这个脚本

## 输出
- 完整的脚本SOP文件（`.md`）
- 文件保存到 `~/.openclaw/skills/general-scripts/`

## 依赖
- 无（只需要用户提供脚本功能描述即可）

## 步骤

1. **确认脚本信息**
   - 确认脚本名称
   - 确认脚本功能描述
   - 确认触发条件

2. **按照六要素结构撰写**
   ```
   ## 触发条件
   [描述什么时候需要运行这个脚本]

   ## 执行方式
   [描述是创建子代理执行还是当前会话执行]

   ## 输入
   [列出需要哪些输入参数]

   ## 输出
   [列出脚本会输出什么]

   ## 依赖
   [列出依赖哪些其他脚本或技能]

   ## 步骤
   [分步骤列出执行顺序，每一步做什么]
   ```

3. **检查格式**
   - 确认六个部分都完整：触发条件、执行方式、输入、输出、依赖、步骤
   - 确认步骤编号正确
   - 确认markdown格式正确

4. **保存文件**
   - 文件名：`脚本名称.md`（全小写，用中划线分隔空格）
   - 路径：`~/.openclaw/skills/general-scripts/`
   - 设置正确文件权限

5. **验证保存**
   - 检查文件是否存在
   - 检查文件内容是否完整

6. **更新 SKILL.md**
   - 在 `general-scripts/SKILL.md` 中添加这个脚本的描述
   - 提交到Git

7. **记录到工作日志**
   - 记录创建时间、脚本名称、路径
