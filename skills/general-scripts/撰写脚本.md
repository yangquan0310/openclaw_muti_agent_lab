# 撰写脚本SOP

## 触发条件
需要创建新的操作脚本

## 输入
- 脚本名称：脚本的名称
- 脚本功能描述：脚本要完成什么任务
- 触发场景：什么时候需要触发这个脚本

## 依赖
- 无（只需要用户提供脚本功能描述即可）

## 步骤

1. **确认脚本需求**
   - 明确脚本命名
   - 明确脚本功能
   - 明确触发条件

2. **撰写脚本内容**
   
   按照五要素结构撰写：
   
   ```
   ## 触发条件
   [描述什么时候需要运行这个脚本]

   ## 依赖
   [列出依赖哪些其他脚本或技能]

   ## 输入
   [列出需要哪些输入参数]

   ## 步骤
   [标题采用动宾结构，明确动作链条]
   [内容、规范、原则等具体要求/信息通过列表格式写在下方]

   ## 输出
   [列出脚本会输出什么]
   ```

3. **检查脚本格式**
   - 检查五要素完整性
   - 检查步骤编号

## 输出
- 完整的脚本SOP流程
