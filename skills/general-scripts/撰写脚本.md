# 撰写脚本SOP

## 触发条件
需要创建新的操作脚本，按照**触发条件、执行方式、输入、输出、依赖、步骤**六要素格式撰写

## 执行方式
当前会话执行，根据用户提供的脚本功能描述撰写完整SOP

**重要：如果是子代理执行，必须提供完整的上下文，包括：**
1. **技能** - 子代理需要使用的技能列表
2. **密钥** - 所需的API密钥或访问凭证
3. **使用方法** - 技能的具体使用方法和示例
4. **任务要求** - 明确的任务目标和预期结果
5. **执行方案/步骤** - 详细的执行步骤和流程

## 输入
- 脚本名称：脚本的名称
- 脚本功能描述：脚本要完成什么任务
- 触发场景：什么时候需要触发这个脚本

## 输出
- 完整的脚本SOP文件（`.md`）
- 文件保存到 `~/.openclaw/skills/general-scripts/`

## 依赖
- 无（只需要用户提供脚本功能描述即可）

## 步骤

1. **确认脚本需求**
   - 明确脚本命名
   - 明确脚本功能
   - 明确触发条件

2. **撰写脚本内容**
   
   按照六要素结构撰写：
   
   ```
   ## 触发条件
   [描述什么时候需要运行这个脚本]

   ## 执行方式
   [描述是创建子代理执行还是当前会话执行]
   
   **如果是子代理执行，必须提供完整的上下文：**
   - 技能：[子代理需要使用的技能列表]
   - 密钥：[所需的API密钥或访问凭证]
   - 使用方法：[技能的具体使用方法和示例]
   - 任务要求：[明确的任务目标和预期结果]
   - 执行方案/步骤：[详细的执行步骤和流程]

   ## 依赖
   [列出依赖哪些其他脚本或技能]

   ## 输入
   [列出需要哪些输入参数]

   ## 步骤
   [标题采用动宾结构，明确动作链条]
   [内容、规范、原则等具体要求/信息通过多级标题写在下方]
   
   **如果是子代理执行，步骤中必须包含：**
   - 传递完整上下文给子代理
   - 明确子代理的任务边界
   - 规定子代理的输出格式
   - 主代理接收和验证子代理结果

   ## 输出
   [列出脚本会输出什么]
   ```

3. **检查脚本格式**
   - 检查六要素完整性
   - 检查子代理上下文
   - 检查步骤编号
   - 检查markdown格式

4. **保存脚本文件**
   - 确定文件名称
   - 确定存储路径
   - 设置文件权限

5. **验证文件完整性**
   - 验证文件存在
   - 验证内容完整

6. **更新技能索引**
   - 更新SKILL.md
   - 提交到Git

7. **记录执行日志**
   - 记录创建时间
   - 记录脚本名称
   - 记录存储路径
