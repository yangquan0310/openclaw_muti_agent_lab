# 通用脚本技能

提供大管家的标准化操作脚本集合，支持文档管理、项目管理、日志记录等全生命周期操作。

## 功能特性

- ✅ **模块化拆分**：每个操作一个独立脚本，按需调用
- ✅ **标准化脚本格式**：所有脚本遵循统一的六要素SOP结构（触发条件、执行方式、依赖、输入、输出、步骤）
- ✅ **智能决策执行**：根据具体场景自动选择执行路径
- ✅ **SOP流程规范**：严格遵循标准操作流程规范
- ✅ **多平台支持**：支持飞书、腾讯云文档等多种云平台

## 脚本列表

### 基础操作脚本
| 脚本名称 | 文件名 | 描述 | 用途 |
|---------|--------|------|------|
| 写入项目元数据 | `写入项目元数据.md` | 创建并写入项目元数据.json，包含项目信息和云文档映射结构 | 新项目初始化 |
| 创建工作日志 | `创建工作日志.md` | 创建工作日志的标准流程 | 任务完成后记录执行日志 |
| 创建新项目 | `创建新项目.md` | 创建新项目目录结构的标准流程 | 初始化新研究项目 |
| 撰写脚本 | `撰写脚本.md` | 按照六要素SOP规范撰写新脚本 | 创建新的通用脚本 |
| 创建文件 | `创建文件.md` | 创建新的文档文件并按规范保存到项目目录 | 创建新文档 |
| 修改文件 | `修改文件.md` | 修改已存在的文档文件，备份旧版本并更新版本号 | 修改现有文档 |
| 映射云文档 | `映射云文档.md` | 维护本地文件与云文档的映射关系到项目元数据 | 云文档映射管理 |
| 控制版本 | `控制版本.md` | 备份旧版本，升级版本号，管理文档版本 | 文档版本控制 |
| 管理知识库 | `管理知识库.md` | 将检索到的文献存入项目知识库的index.json | 文献索引管理 |
| 检索文献 | `检索文献.md` | 使用Semantic Scholar和Zotero检索文献并更新项目知识库 | 文献检索与入库 |

## 详细说明

### 1. 创建工作日志脚本

**功能概述**：
标准化的日志记录流程，确保任务执行的完整追溯。

**使用场景**：
- 任务执行完成后记录执行情况
- 记录任务参与者和分工说明
- 保存生成文件的存储路径
- 记录任务执行过程中的问题与解决方案

**输出格式**：
```
路径: ~/实验室仓库/日志文件/YYYY-MM-DD/HH-MM-SS-[agents]-[任务类型].md
```

### 2. 创建新项目脚本

**功能概述**：
标准化的项目初始化流程，确保项目结构规范统一。

**使用场景**：
- 初始化新的研究项目
- 创建项目目录结构
- 生成项目元数据文件
- 创建项目说明文档

**目录结构**：
```
YYYY-MM-DD_项目名/
├── 文档/           # 用户上传文档
├── 草稿/           # 论文草稿
├── 终稿/           # 终稿
├── 知识库/         # 项目专属知识库
├── 元数据.json     # 项目元数据
└── README.md       # 项目说明
```

### 3. 查询规范脚本

**功能概述**：
标准化的规范查阅流程，确保操作符合实验室规章制度。

**使用场景**：
- 需要查阅记忆管理规范
- 需要查阅本地文件管理规范
- 需要查阅云文档管理规范
- 需要查阅日志管理规范
- 需要查阅更新管理规范

**支持的规范**：
- 记忆管理规范.md
- 本地文件管理规范.md
- 云文档管理规范.md
- 日志管理规范.md
- 更新管理规范.md

## 脚本格式规范

所有脚本文件遵循以下标准格式：

```
**触发条件**：(触发条件内容)
**执行方式**：(直接执行 / 创建子代理)
**依赖**：(依赖哪些其他脚本或技能)
**输入**：
**输出**：
**步骤**：
 - 步骤1
 - 步骤2
```
