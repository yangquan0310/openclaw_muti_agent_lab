# S2-总结文献笔记

## 触发条件
需要对知识库中特定文献进行深度阅读并提取关键信息，形成结构化笔记

## 依赖
- 项目知识库index.json
- Python处理JSON文件

## 输入
- 目标文献列表（从知识库筛选）
- 项目路径
- 笔记文件名

## 步骤

### 1. 读取目标文献
- 从项目知识库index.json中筛选需要总结的文献
- 确定文献范围（可按照重要性、类型、主题等筛选）

### 2. 整理文献
**排除非期刊文献**：
- 预印本（preprint）
- 手稿（manuscript）
- 学术论文

**排除无法提取文献**：
- abstract字段为空

**逐篇提取关键信息**
- 逐篇阅读文献摘要（必须一篇一篇阅读，禁止批量处理） 理解研究背景与核心内容
- 用中文凝练核心要点（禁止直接复制英文原文）：
- 📊实证文献必须包含：
    - 研究问题：文献试图解决什么科学问题（中文概括，1-2句话）
    - 研究方法：采用什么方法、被试/样本情况、实验设计（中文概括，1-2句话）
    - 研究结果：发现了什么主要结果、效应大小（中文概括，1-2句话）
    - 研究结论：得出什么结论、理论贡献（中文概括，1-2句话）
- 📖综述文献必须包含：
    - 研究问题：文献试图解决什么科学问题（中文概括，1-2句话）
    - 研究结果：发现了什么主要结果、效应大小（中文概括，1-2句话）
    - 研究展望：未来可以进一步解决什么问题（中文概括，1-2句话）
- 💡理论文献必须包含：
    - 研究问题：文献试图解释什么科学问题（中文概括，1-2句话）
    - 理论观点：用什么概念，概念逻辑是什么，怎么解释科学问题（中文概括，1-2句话）


### 3. 生成笔记文件
- 按照知识库示例 `/root/.openclaw/scripts/检索文献/知识库索引示例.md`按照创建JSON格式笔记文件
- 保存到`知识库/笔记/{笔记}.json`

### 4. 管理元数据
按照「管理元数据.md」更新项目元数据.json中的笔记文件路径

## 输出
- 笔记文件：`知识库/笔记/{笔记}.json`
- 本地路径、版本信息
- 返回主代理：文件路径、文献统计信息
