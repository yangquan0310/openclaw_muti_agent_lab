## 工作流

### 工作流1：技能习得（新技能引入）

**触发条件**：发展日记记录到全新能力的掌握

**步骤**：
1. **识别新技能**
   - 从发展日记中提取新习得的能力
   - 判断是否为结构性新技能（vs 临时技巧）

2. **技能结构化**
   - 定义技能名称和描述
   - 明确输入参数和输出结果
   - 编写标准 SKILL.md 规范（参考 `Skill-developer`）
   - 如需代码，使用面向对象框架编写

3. **注册技能**
   - 添加到 Agent 个人技能文件夹 `workspace/<agent>/skills/`
   - 更新 `_meta.json`
   - 在根 SKILL.md 中添加路由（如需要）
   - **更新 `skills/README.md` 技能索引**

4. **验证与测试**
   - 确认技能可独立调用
   - 确认文档与实现一致

5. **记录更新日志**

### 工作流2：技能细化（同化）

**触发条件**：发展日记显示某技能使用方式得到优化

**步骤**：
1. **识别优化点**
   - 连续多次使用改进方法成功
   - 用户明确反馈某流程更高效

2. **更新技能文档**
   - 修改对应 SKILL.md 中的工作流
   - 更新参数说明
   - 更新模板和示例
   - **如技能名称/描述变更，同步更新 `skills/README.md` 索引**

3. **同步版本号**
   - 遵循语义化版本（major.minor.patch）
   - 细化更新通常提升 minor 或 patch

4. **记录更新日志**

### 工作流3：技能淘汰（顺应）

**触发条件**：发现某技能不再适用或已被替代

**步骤**：
1. **评估淘汰必要性**
   - 确认技能已过时
   - 确认有替代方案

2. **标记弃用**
   - 在 SKILL.md 顶部添加弃用声明
   - 指向替代技能路径
   - **更新 `skills/README.md` 索引，标记为 deprecated**

3. **迁移依赖**
   - 检查其他技能是否依赖此技能
   - 更新依赖关系到新版本或替代技能

4. **归档或删除**
   - 轻度淘汰：保留文件，标记 deprecated
   - 重度淘汰：移动至 `archive/` 目录

5. **记录更新日志**

## 使用指南

### 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `operation` | string | ✅ | 操作类型：`acquire` / `refine` / `deprecate` |
| `skill_name` | string | ✅ | 技能名称 |
| `skill_description` | string | ✅ (acquire) | 技能描述 |
| `evidence` | object | ✅ | 支持本次更新的经验证据（来自日记） |

### 输出结果

| 输出项 | 格式 | 说明 |
|--------|------|------|
| `skill_spec` | Markdown | 技能的 SKILL.md 规范文档 |
| `update_log` | Markdown | 技能更新记录 |
| `migration_guide` | Markdown | 依赖迁移指南（deprecate 时） |
